501 - Layer 5 Entry — iChoCh & SB Confirmation

TL;DR — Layer 5 entry confirmation reads a DAG snapshot where phase == "D" and phase_sub_status == "watch". The DAG uses only last_sc (macro ChoCh, SC01/SC02) for all structural transitions — iChoCh (last_isc, SC06) and counter SB are Layer 5 signals. Path A: ltf_counter_choch_seen (SC06 iChoCh) — finer bounce-failure signal at confirmed ITR pivot. Path B: ltf_counter_sb_seen (macro counter SB, pullback_confirmed) — macro BOS after structurally quiet bounce. SL reference: phase_d_shadow.choch_1.level. Quality grading: phase_d_shadow.pro_attempt. Both signals are available as EC debug facts from the ltf_counter_choch candidate — no new EC method required.
→ Related: 402 Phase D → Related: 402 Phase C

00 DAG / Layer 5 Boundary

The DAG (Layer 4) owns phase state transitions. It uses only macro structural events (last_sc, SC01–SC04) because these fire reliably against the wick-defined range boundary and are structurally unambiguous. iChoCh (SC05/SC06, last_isc) fires against confirmed internal ITR pivot levels — earlier and finer, but not a DAG gate.

Layer 5 reads the snapshot after the DAG has committed phase. At D.watch, the DAG is holding — the bounce started but no counter MSS has fired yet to force C.pullback. Layer 5 reads the same EC candidate facts to decide if iChoCh or a macro SB is present and whether that is enough confirmation to enter. 

DAG (Layer 4) uses:
  last_sc  (SC01/SC02/SC03/SC04)  — macro ChoCh and SB, wick range boundary
  ← all DAG phase transitions gated on last_sc only

Layer 5 uses additionally:
  last_isc (SC05/SC06)            — internal iChoCh, confirmed ITR pivot break
  ltf_counter_choch_seen          — EC debug fact (from ltf_counter_choch candidate)
  ltf_counter_sb_seen             — EC debug fact (from same candidate)

EC ltf_counter_choch candidate emits both signals in the same debug_facts dict.
Layer 4 and Layer 5 read different keys from the same candidate.
The boundary is not about signal strength — it is about responsibility. The DAG holds structural commitment memory; Layer 5 holds entry permission state. iChoCh is an entry confirmation signal, not a phase-advance signal.
Terminology LTF structural signals 
LTF iChoCh  = LTF internal pivot break  — SC06 (or SC05 LTR variant, disabled)
                last_isc, is_internal=True
                fires when LTF close breaks a confirmed PE03/PE04 ITR pivot level
                finer signal: visible before the macro range boundary is reached
                source_store == "structure_isc"

LTF ChoCh   = LTF macro structural flip — SC01/SC02, last_sc, choch=True
                fires against wick-defined range boundary
LTF SB      = LTF structural break      — SC01/SC02, last_sc, choch=False  (BOS)
BoS / MSS   = orderflow engine events   — separate lineage, not from structureEngine

01 Path A — iChoCh Entry Confirmation

The bounce has failed: the LTF closed through a confirmed ITR pivot low (bullish HTF) or pivot high (bearish HTF) in the counter-HTF direction. SC06 (last_isc) fires before a full macro ChoCh — it is the finer signal. Path A consumes this as entry confirmation.

Path A gate Layer 5 — D.watch snapshot 
DAG snapshot prerequisite:
  phase == "D"
  phase_sub_status == "watch"
  direction == expected long | short

EC debug fact (from ltf_counter_choch candidate):
  ltf_counter_choch_seen == True
    ← SC06 itrIChoCh, last_isc
    ← last_isc.breakDirection == counter-HTF direction
    ← is_internal == True, source_store == "structure_isc"
    ← fires against confirmed PE03 (bearish pivot high) or PE04 (bullish pivot low)

Freshness floor:
  ltf_counter_choch_event_at > phase_d_shadow.watch_entered_at
  ← prevents consuming a stale iChoCh that pre-dates D.watch open

SL reference:
  phase_d_shadow.choch_1.level     ← pro-HTF SC that opened D.watch
  ← fallback if no per-trade zone is available

Available EC debug fields at this gate:
  ltf_counter_choch_event_at        ← event timestamp
  ltf_counter_choch_level           ← ChoCh floor price
  ltf_counter_choch_event_id        ← event ID for provenance
  ltf_counter_choch_source_level_id ← ITR pivot level that was broken
  ltf_counter_choch_source_store    "structure_isc"
Path A signals before the macro range boundary is reached. This is its primary advantage: the LTF pivot break gives earlier entries than waiting for a full macro ChoCh. The trade-off is sensitivity to pivot level noise.

02 Path B — Macro SB Entry (pullback_confirmed)

Path B handles the case where D.watch ran without SC06 firing — the bounce was structurally quiet (no ITR pivot break), but the LTF structure engine recorded the bounce via ITR/SD pivot recognition and entered pullback_confirmed. The next counter macro SB (BOS continuation, choch=False) then confirms the pullback is resuming.

Path B gate Layer 5 — D.watch snapshot 
DAG snapshot prerequisite:
  phase == "D"
  phase_sub_status == "watch"
  direction == expected long | short

EC debug fact (from ltf_counter_choch candidate):
  ltf_counter_sb_seen == True
    ← SC01/SC02 last_sc, choch=False, break_direction == counter-HTF direction
    ← LTF structure engine phase == "pullback_confirmed"
    ← LTF BOS: counter range floor extended, no bias flip
    ← source_store == "structure_sequence"

Note — Path B is mutually exclusive with Path A in the EC compiler:
  sb_seen = not choch_seen AND structure_sb AND correct direction
  if choch_seen is True, sb_seen is always False

Freshness floor:
  ltf_counter_sb_event_at > phase_d_shadow.watch_entered_at

SL reference:
  ltf_counter_sb_level              ← macro SB floor price
  ← fallback: phase_d_shadow.choch_1.level

Available EC debug fields at this gate:
  ltf_counter_sb_event_at           ← event timestamp
  ltf_counter_sb_level              ← SB floor price
  ltf_counter_sb_event_id
  ltf_counter_sb_source_level_id
  ltf_counter_sb_source_store       "structure_sequence"
! Path B is a macro-level signal (not iChoCh) but remains Layer 5 territory — the DAG does not act on counter SB. The DAG only advances from D.watch when the orderflow MSS fires (LTF anchor swing low broken). Path B handles the case where that MSS has not fired yet but the LTF structural quiet bounce is clear enough for entry.

03 Quality Context Fields

Layer 5 reads three fields from debug_facts and shadow state to grade trade quality. None of these are gate conditions — they affect sizing and SL placement, not whether an entry is taken.

choch_1 — SL and provenance phase_d_shadow 
phase_d_shadow.choch_1:
  trigger_type    "choch"
  at              eventTimestamp of the pro-HTF SC01/SC02 that opened D.watch
  level           price level of that SC (e.g. prior range_high for bullish long)

Layer 5 reads:
  choch_1.level   ← static SL reference when no per-trade SD zone is available
  choch_1.at      ← freshness anchor for Path A/B freshness floor (watch_entered_at)
pro_attempt — trade quality grading phase_d_shadow 
phase_d_shadow.pro_attempt (accumulated while holding D.watch each bar):
  htf_reaction_status   ← htf_counter_reaction EC status ("ready" | "collecting" | None)
  ltf_story_status      ← ltf_counter_story EC status ("ready" | "collecting" | None)

Quality grading (Layer 5 reads):
  htf_reaction_status == "ready"   → meaningful HTF reaction — tighter SL, fuller size
  ltf_story_status    == "ready"   → clean LTF counter structure — stronger confirmation
  both None                        → shallow grind — acceptable, lower quality
phase_c_origin_node — C phase quality debug_facts 
Available when phase == "C" (phase_sub_status == "pullback" | "pullback_weaken"):
  phase_c_origin_node:
    "D.watch_mss"                  ← counter MSS broke anchor after D.watch: structural confirmation
    "D.speculation_mss"            ← legacy (D.speculation removed from DAG)
    "E.pullback_developing_no_pro" ← V1 fast path: pullback depth ≥ 51%, no D.watch ever

Quality interpretation:
  "D.watch_mss"                  → highest quality: D.watch held + MSS broke anchor
  "E.pullback_developing_no_pro" → lower quality: no pro-attempt seen, depth threshold only

04 Phase C Entry

When the DAG advances to C.pullback or C.pullback_weaken, Layer 5 reads the C-phase snapshot to time entry. The DAG moves from D.watch to C.pullback on counter MSS — structural confirmation that the pullback is resuming. Layer 5 still needs iChoCh (SC06) for precise entry timing within the C phase.

C.pullback entry Layer 5 — C phase snapshot 
DAG snapshot:
  phase == "C"
  phase_sub_status == "pullback"   ← or "pullback_weaken"
  direction == expected long | short

Layer 5 entry signal (same as Path A, now in C context):
  ltf_counter_choch_seen == True   ← SC06 iChoCh counter-HTF direction
  freshness: ltf_counter_choch_event_at > phase_c_shadow.entered_at

phase_c_origin_node available for quality grading (see §03):
  "D.watch_mss"                  → structural confirmation path
  "E.pullback_developing_no_pro" → depth threshold path

C.pullback_weaken context:
  phase_sub_status == "pullback_weaken"
  ← pro-HTF last_sc broke a pullback lower-high (LH broken upward in bullish HTF)
  ← bounce attempt within the counter pullback — higher-quality entry zone
  ← Layer 5 may treat pullback_weaken bars as priority entry context
C.pullback_weaken does not mean the trade is invalid — it means the counter pullback had an internal pro-HTF bounce attempt. This is often the best entry zone: the pullback is continuing but showed strength, marking a refined POI level.

05 Algorithm

Layer 5 reads the DAG snapshot bar-by-bar. It tracks its own entry state (pending limit, filled attempts, budget) separately from the DAG. The DAG snapshot is the clock; Layer 5 applies permission rules on each bar. 

# Layer 5 reads from DAG snapshot each bar
phase              = snapshot["hypothesis"]["phase"]
phase_sub_status   = snapshot["hypothesis"]["phase_sub_status"]
direction          = snapshot["hypothesis"]["direction"]
debug_facts        = snapshot["hypothesis"]["debug_facts"] or {}

# Shadow fields (available via debug_facts keys)
watch_entered_at   = debug_facts.get("phase_d_shadow_watch_entered_at")
choch_1_level      = debug_facts.get("phase_d_shadow_choch_1_level")
pro_attempt        = debug_facts.get("phase_d_shadow_pro_attempt") or {}

# EC facts from ltf_counter_choch candidate (also in debug_facts)
choch_seen         = debug_facts.get("ltf_counter_choch_seen", False)
choch_event_at     = debug_facts.get("ltf_counter_choch_event_at")
choch_level        = debug_facts.get("ltf_counter_choch_level")
sb_seen            = debug_facts.get("ltf_counter_sb_seen", False)
sb_event_at        = debug_facts.get("ltf_counter_sb_event_at")
sb_level           = debug_facts.get("ltf_counter_sb_level")

if phase == "D" and phase_sub_status == "watch":
    # Path A check (§01)
    if choch_seen and choch_event_at and watch_entered_at:
        if choch_event_at > watch_entered_at:
            sl_ref = choch_1_level
            # → enter with Path A confirmation
    # Path B check (§02)
    elif sb_seen and sb_event_at and watch_entered_at:
        if sb_event_at > watch_entered_at:
            sl_ref = sb_level or choch_1_level
            # → enter with Path B confirmation
# Path A — iChoCh gate detail
# choch_seen: SC06 itrIChoCh, last_isc, break_direction == counter-HTF
# source: _compile_ltf_counter_choch() in evidence_compiler.py

choch_seen = bool(
    _structure_ichoch_seen(last_isc)              # eventAction == "structure_ichoch" or structure_ichoch == True
    and last_isc.get("breakDirection") == expected_counter_break
)

# Freshness check prevents stale pre-D.watch iChoCh from firing
is_fresh = (
    choch_event_at is not None
    and watch_entered_at is not None
    and choch_event_at > watch_entered_at
)

# SL resolution (Path A)
sl = choch_level or choch_1_level  # ChoCh floor first, pro-HTF ChoCh as fallback
# Path B — macro SB gate detail
# sb_seen: SC01/SC02 last_sc, choch=False, break_direction == counter-HTF
# mutually exclusive with Path A in the EC compiler

sb_seen = bool(
    not choch_seen
    and _structure_event_action(last_sc) == "structure_sb"   # choch=False, BOS only
    and last_sc.get("breakDirection") == expected_counter_break
)

# pullback_confirmed check (LTF structure phase — in the snapshot)
ltf_phase = snapshot.get("lower_structure", {}).get("phase")
pullback_confirmed = ltf_phase == "pullback_confirmed"

# Combine: SB fires only after the bounce has been recorded structurally
path_b_valid = sb_seen and pullback_confirmed

# Freshness check
is_fresh = (
    sb_event_at is not None
    and watch_entered_at is not None
    and sb_event_at > watch_entered_at
)

# SL resolution (Path B)
sl = sb_level or choch_1_level  # BOS floor first, pro-HTF ChoCh as fallback

06 Open Issues

Next Milestone

Layer 5 module not yet implemented. This note documents the intended Layer 5 contract derived from DAG shadow state. The actual EntryProtocol, ControlledChase, and BacktestBrokerAdapter modules are pending (see 512-entry-protocol-algo.md).

Edge Case

Freshness floor relies on timestamp comparison. watch_entered_at is the eventTimestamp of choch_1, not cursor_time. If the SC event timestamp is slightly mismatched against the LTF bar close time, Path A may miss a valid iChoCh on the exact opening bar of D.watch.

Edge Case

Path A and Path B are mutually exclusive in the EC compiler. If SC06 iChoCh fires, sb_seen is forced to False. There is no scenario where both fire in the same bar. This is correct behavior — document as an invariant when Layer 5 is implemented.

Out of Scope

SC07/SC08 LTR variants disabled. SC07 (ITR range LTR ChoCh) and SC08 (ITR range LTR SB) are disabled in the current structureEngine. Path A uses SC06 only. If LTR is re-enabled, the iChoCh gate would need to cover both SC06 and SC07.

Archived · Moved

D.speculation mechanics. Path A and Path B were formerly DAG gates that stamped choch_2 and advanced to D.speculation. Removed from DAG in commit 10dff12. Mechanics documented here (Layer 5 territory). choch_2 and speculation_entered_at fields removed from PhaseDShadow.

Archived · Moved

pro_attempt_seen trigger changed from last_isc to last_sc. Previously D.watch opened on SC06 iChoCh (primary) with SC01/SC02 as squeeze-edge-case fallback. After iChoCh → Layer 5 migration, pro_attempt_seen uses last_sc only. The squeeze edge case is no longer special — all D.watch opens use macro ChoCh.
