00 Migration Boundary

This tracker keeps the open decisions visible while Layer 4 is rebuilt phase by phase. The target boundary is simple: EC emits facts, the DAG selects the phase node, and each phase shadow stores only the memory needed inside that phase.

Open migration rule: do not let a shadow thesis decide the global phase. Shadow state should explain the current phase's internal status, while the DAG remains responsible for phase transitions.
Next Milestone

Define phase-owned shadow objects. Decide whether each phase gets a small dataclass such as PhaseEShadow, DShadow, and BShadowThesis, instead of one generic ShadowThesis that quietly means B.

Edge Case

Stop legacy priority from leaking back in. The old classifier searched E, D, C, B, A by priority. The migration needs an explicit rule for when that priority is still valid, and when the DAG must hold the active node.

Fallback

Prefer one active phase during rebuild. If cross-phase behavior becomes noisy, run the classifier with only E enabled first, then add D, C, B, and A back in order.

01 none / warmup

none is still the silence state, not a trading phase. The open work is making sure it only means warmup, no closed HTF structure, or genuinely no trade, instead of catching valid post-warmup phase states.

Open Issue

Allow warmup to self-locate into any current phase. Once the first usable HTF structure exists, the DAG should be able to move from none(warmup) into E, D, C, B, or A from current EC/DAG evidence instead of only supporting none → E.seeking.

Edge Case

Separate silence reasons. Keep warmup silence, missing 15m/4H close, no structure, and no trade as distinct none_sub_status values so a valid E origin does not disappear under the catch-all.

Fallback

Debug blocked transitions. If a bar stays none while EC has enough E facts, emit a blocked-transition reason such as blocked_none_to_e.

02 Phase E · Closed

Phase E migration is closed. Price remains in expansion monitoring while the HTF P/D level is extending, stalling, or developing a clean counter pullback. Only an explicit destination-phase contract may move the DAG out of E.

The completed ownership split is explicit. EC owns observable facts, Layer 4 owns phase choice, and the Phase E shadow thesis owns only the internal E.seeking / E.stalling / E.pullback_developing memory.

Migration status: Closed. There are no remaining Phase E migration items in this ledger. References to E inside none, D, and C describe the origin boundary for those destination phases; they do not reopen Phase E migration. Lower-risk Phase E runtime oddities remain documented as edge cases in 402 Phase E.

  - whether current hypothesis phase is none, E, D, C, B, or A
  - none/warmup -> current best phase transition once usable HTF structure exists
    (E, D, C, B, or A)
  - holding E when LTF probes outside PD range before HTF close
  - E -> D transition only on the agreed counter-reaction contract
  - E.stalling -> E.pullback_developing when LTF counter orderflow MSS / P-D flip appears

Phase E shadow thesis must not own:
  - global phase selection
  - HTF/LTF fact detection
  - D/C/B/A transition priority
  - raw snapshot reads

04 Phase C · In Progress

2-state model: C.pullback / C.pullback_weaken. Origin tracked via phase_c_origin_node metadata. DAG uses MSS (macro ChoCh last_sc) for all structural transitions; iChoCh reserved for Layer 5. 

C.pullback
  + pro-HTF MSS breaks pullback LH (last_sc, pro-HTF direction)  → C.pullback_weaken
  + ltf_pullback_depth_pct >= 50%                                 → B.xxx
  + new HTF extreme                                               → E.seeking

C.pullback_weaken
  + counter-HTF MSS re-fires in pullback direction (LL break)     → C.pullback
  + new HTF extreme                                               → E.seeking
  → B.xxx  (deferred, decide during Phase B)

05 Phase B

B is currently the only extracted shadow thesis, but that object is really B-specific. The clean migration path is to gate it off, rename or reshape it, and bring it back after E, D, and C are explicit.

Next Milestone

Disable B while rebuilding earlier phases. Add a config or classifier gate so B cannot take priority while E, D, and C contracts are being verified in the replayer.

Next Milestone

Rename generic ShadowThesis. If the fields remain B-only, rename it to BShadowThesis or split it into phase-owned state before adding D/C/A shadow logic.

Edge Case

Split initiation from shallow reclaim. The B migration still needs a crisp boundary between the B setup candidate and the shallow-reclaim memory that survives across bars.

Fallback

Reset rules need proof in replay. B shadow should define exactly when it survives, resets, or gets consumed by A, with debug fields showing the reason.

06 Phase A

A should be last because it is the trade-objective phase. The open question is not just how to detect A, but how to prove it came from the right upstream phase.

Next Milestone

Migrate _phase_a_setup() off raw snapshots. A setup needs an EC candidate or an explicit deferral note before it can be trusted inside the DAG.

Edge Case

Define A origin from B. The DAG should reject an A candidate that does not have a valid B lineage, unless a separate exceptional origin is deliberately defined.

Fallback

Revalidate A finale after B returns. Even if _phase_a_finale() already has EC wiring, it should be replay-checked after B shadow rules are renamed and gated back in.
