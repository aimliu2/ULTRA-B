Layer 4 · Hypothesis Classifier · Phase C

Phase C — Counter Pullback

TL;DR — Phase C is the counter-pullback phase. Two DAG states: C.pullback and C.pullback_weaken. Entry origin tracked via phase_c_origin_node metadata (not new states). All structural transitions use MSS (last_sc); iChoCh (last_isc) is Layer 5 territory. V1: E.pullback_developing + no D + depth ≥ 51%. V2: D.watch + counter MSS (LL break). Weaken: pro-HTF MSS breaks pullback LH. Recovery: counter MSS re-fires.

00 Core Concept

Phase C names the counter-trend pullback after the Phase E expansion leg stops being clean continuation. It is not the first reaction itself. It is the counter story that follows the reaction, with enough LTF structure to either watch for a counter POI or arm against one.

The classifier keeps the Phase E direction as prior_phase_e_direction, then Phase C trades the opposite direction. If Phase E was bullish, C is short. If Phase E was bearish, C is long.

Discretionary read chart logic

C is the moment where the pullback has a structure story of its own. The chart is no longer only saying "E stopped expanding"; it is saying "the counter leg has enough internal evidence to be tracked." 

Phase E bullish:
  E expansion up
  reaction / stall / pullback begins
  LTF counter story turns bearish
  Phase C direction = short

Phase E bearish:
  E expansion down
  reaction / stall / pullback begins
  LTF counter story turns bullish
  Phase C direction = long
Phase C is counter to the prior Phase E direction. It can be watching with a story but no selected POI, or armed when a selected counter POI exists.

01 EC Inputs

The current Phase C setup is deliberately thin. Layer 4 calls _ec_candidate(snapshot, "ltf_counter_story") and reads the resulting facts. The EC decides whether the counter story is ready; the DAG decides which C node name that story belongs to.

EC package ltf_counter_story 
Gate:
  c_gate.status == "ready"  → phase_c_story_ready

Facts consumed by Layer 4:
  selected_poi
  ltf_counter_sd_zone_ids
  ltf_counter_sd_returned_zone_ids
  ltf_bias_counter_htf
  ltf_counter_pd_break
  ltf_counter_pullback_confirmed
Status and entry policy Layer 4 output 
story_ready + no selected_poi:
  phase = C
  status = watching
  entry_policy = wait

story_ready + selected_poi:
  phase = C
  status = armed
  entry_policy = hybrid only if pullback trading is enabled

otherwise:
  no Phase C promotion from this gate
EC should report counter evidence. It should not decide whether the node is C.hard_pullback, C.slow_pullback, or C.htf_reaction_pullback. That naming belongs to Layer 4 because it depends on DAG origin.

02 State Machine

Phase C uses two DAG states. Entry origin is tracked via phase_c_origin_node metadata — it does not add new states. All structural transitions use MSS (last_sc, macro ChoCh). iChoCh (last_isc) is reserved for Layer 5 entry confirmation.

C.pullback active state 
C.pullback
  + pro-HTF MSS breaks pullback LH
    (bullish HTF: last_sc fires pro-HTF direction = LH broken upward)
  → C.pullback_weaken

C.pullback
  + ltf_pullback_depth_pct >= 50%
  → B.xxx

C.pullback
  + new HTF extreme fires
  → E.seeking  (rare — extreme FVG expansion; C.pullback_weaken usually catches
                the pro-attempt first before HTF extreme is confirmed)

Debug fields: phase_c_origin_node, phase_sub_status = "pullback"

C.pullback_weaken active sub-state

Enters when the counter pullback shows a pro-HTF MSS (LH break). The recovery condition is: the pro-attempt within the pullback itself fails — counter MSS re-fires in the pullback direction (LL break). Not "LL confirmed"; not "floor tested." The same MSS mechanic that opened C.pullback fires again. 

C.pullback_weaken
  + counter-HTF MSS re-fires in pullback direction (LL break)
  → C.pullback

C.pullback_weaken
  + new HTF extreme fires
  → E.seeking

C.pullback_weaken
  → B.xxx  (deferred — decide during Phase B)

Debug fields: phase_sub_status = "pullback_weaken", phase_c_weaken_at

Archive — old 5-variant model superseded 2026-06-10

The old model named five Phase C variants. Retired in favour of the 2-state model above.

C.htf_reaction_pullback   — D reaction → C (D.HTF_reaction_point etc.)
C.hard_pullback           — E fast path, depth >= 50%, tight P/D range
C.slow_pullback           — E.pullback_developing + ltf_counter_bos_confirmed (Case F)
C.pullback.no_followthrough  — B.initiation_watch failure
C.pullback.after_inducement  — B.initiation_watch rejection

In the new model: D-to-C and E-to-C paths are entry origins stored in phase_c_origin_node, not separate states. B failure → C is deferred (V4). _phase_c_sub_status_from_current_d() is eliminated.

03 Transitions

Two entry paths. Origin is recorded in phase_c_origin_node and does not create new states.

V1 — E.pullback_developing, no D, depth ≥ 51% E → C 
Condition:
  E.pullback_developing is active
  phase_d_shadow.node == None  (D.watch never opened)
  ltf_pullback_depth_pct >= 51%
  previous_phase not in {"B", "A"}  ← V4 guard

Result:
  → C.pullback
  phase_c_origin_node = "E.pullback_developing_bos"   (Path 2: LTF BoS, slow structural)
  phase_c_origin_node = "E.pullback_developing_depth"  (Path 3: depth >= 51%, fast flip)

Price grinds counter without ever attempting a pro-HTF MSS. D was never entered. Path 2 fires on a fresh counter BoS leg; Path 3 fires when depth reaches 51% (express fallback).

V2 — D.watch + counter MSS breaks LL D → C 
Condition:
  phase_d_shadow.node == "D.watch"
  ltf_counter_orderflow_mss_watch == True  (fresh counter leg, LL break)

Result:
  → C.pullback
  phase_c_origin_node = "D.watch_mss"

Pro-attempt (first pro-HTF MSS) opened D.watch, then the counter MSS breaks LL — pro-attempt definitively failed. Replaces old D.speculation → C.pullback gate.

Exit families C → B / E 
C.pullback        + depth >= 50%          → B.xxx
C.pullback        + new HTF extreme       → E.seeking
C.pullback        + pro-HTF MSS (LH)      → C.pullback_weaken

C.pullback_weaken + counter MSS (LL)      → C.pullback
C.pullback_weaken + new HTF extreme       → E.seeking
C.pullback_weaken                         → B.xxx  (deferred)
The DAG preserves origin via phase_c_origin_node. An E-direct pullback and a D-failed-pro-attempt pullback are not the same historical object, even though they share the same 2-state machine.

04 Algorithm

These snippets show the current classifier shape. Phase C setup is generic; the branch that calls it determines the C sub-status and origin debug fields. 

# classify(), simplified
if previous_phase == "D":
    phase_c = self._phase_c_setup(snapshot, ltf, direction)
    if phase_c["phase_c_story_ready"] and self._phase_d_reaction_confirmed(phase_c):
        hyp = self._phase_c(..., phase_sub_status="htf_reaction_pullback")
        return self._commit(hyp)

phase_d = self._phase_d_setup(...)
if phase_d["phase_d_ready"]:
    return self._commit(self._phase_d(...))

phase_c_from_e = self._phase_c_setup(snapshot, ltf, direction)
fast_hard = self._phase_c_fast_hard_pullback_setup(debug, phase_c_from_e)
if previous_phase != "B" and fast_hard["phase_c_fast_hard_pullback_ready"]:
    hyp = self._phase_c(..., phase_sub_status="hard_pullback")
    return self._commit(hyp)

if previous_phase != "B" and phase_e_shadow_node == "E.pullback_developing":
    if phase_c_from_e["phase_c_ltf_counter_bos_confirmed"]:  # Case F: confirmed_direction + regime directional
        hyp = self._phase_c(..., phase_sub_status="slow_pullback")
        return self._commit(hyp)
def _phase_c_setup(self, snapshot, ltf, prior_direction):
    c_gate, facts = _ec_candidate(snapshot, "ltf_counter_story")
    selected_poi = facts.get("selected_poi")
    story_ready = c_gate.get("status") == "ready"
    armed = bool(story_ready and selected_poi)

    return {
        "phase_c_story_ready": story_ready,
        "phase_c_status": "armed" if armed else ("watching" if story_ready else None),
        "phase_c_direction": "short" if prior_direction == "long" else "long",
        "phase_c_counter_zone_direction": counter_zone_direction,
        "phase_c_selected_poi": selected_poi,
        "phase_c_ltf_bias_counter_htf": bool(facts.get("ltf_bias_counter_htf")),
        "phase_c_ltf_counter_pd_break": bool(facts.get("ltf_counter_pd_break")),
        "phase_c_ltf_counter_pullback_confirmed": bool(facts.get("ltf_counter_pullback_confirmed")),
    }
def _phase_c(self, prior_direction, selected_poi, status, ts, debug, phase_sub_status=None):
    trade_direction = "short" if prior_direction == "long" else "long"
    entry_policy = (
        "hybrid"
        if status == "armed"
        and (self.config.get("allow_pullback_trade")
             or self.config.get("phase_c_allow_pullback_trade"))
        else "skip"
    )
    if status == "watching":
        entry_policy = "wait"

    return Hypothesis(
        phase="C",
        status=status,
        direction=trade_direction,
        swing_alignment="counter_swing",
        internal_alignment="pro_internal",
        phase_sub_status=phase_sub_status,
        entry_policy=entry_policy,
    )
ready = bool(
    debug.get("phase_e_context_status") == "ready"
    and not debug.get("phase_e_context_new_htf_extreme")
    and debug.get("phase_e_context_htf_pd_stopped_expanding")
    and debug.get("phase_e_context_ltf_counter_orderflow_clean")
    and not debug.get("phase_e_context_ltf_counter_orderflow_broken")
    and depth_value is not None
    and depth_value >= 50.0
)

06 Old Model (Archived)

The prior Phase C had five sub-statuses. Superseded by the 2-state C.pullback / C.pullback_weaken model. Preserved here for migration reference.

C.htf_reaction_pullback Archived

The primary D-to-C path. Entered when Phase D confirmed an HTF reaction and the counter story was ready. 

Gate:
  previous_phase == "D"
  phase_c_story_ready                  ← ltf_counter_story EC ready
  _phase_d_reaction_confirmed(phase_c):
    phase_c_ltf_counter_bos_confirmed  ← confirmed_direction == counter + regime directional
    OR phase_c_ltf_counter_pullback_confirmed
    OR ltf_pullback_depth_pct >= depth threshold

DAG:
  phase = C
  phase_sub_status = "htf_reaction_pullback"
  origin: D reaction point → C
C.hard_pullback Archived

Fast E-direct path. Required the P/D range to have stopped expanding and a clean orderflow score with sufficient depth. 

Gate (_phase_c_fast_hard_pullback_setup):
  phase_e_context_status == "ready"
  NOT phase_e_context_new_htf_extreme
  phase_e_context_htf_pd_stopped_expanding == True
  phase_e_context_ltf_counter_orderflow_clean == True
  NOT phase_e_context_ltf_counter_orderflow_broken
  ltf_pullback_depth_pct >= 50.0
  previous_phase not in {"B", "A"}

DAG:
  phase = C
  phase_sub_status = "hard_pullback"
  origin: E fast path, tight P/D range, depth threshold
C.slow_pullback Archived

Slower E-direct path (Case F). Required E.pullback_developing shadow and confirmed orderflow direction. The ltf_counter_bos_confirmed flag demanded both confirmed_direction == counter_ltf_bias and regime == "directional" — meaning the direction scorer had to be non-mixed. This is the gate the orderflow MSS fix (2026-06-11) superseded. 

Gate:
  previous_phase not in {"B", "A"}
  phase_e_shadow_node == "E.pullback_developing"
  phase_c_ltf_counter_bos_confirmed == True
    → confirmed_direction == counter_ltf_bias
      AND regime == "directional"

DAG:
  phase = C
  phase_sub_status = "slow_pullback"
  origin: E.pullback_developing + confirmed orderflow direction
C.pullback.no_followthrough Archived

Re-entry from a failed B initiation. The source ITR anchor was run through — meaning price traced back through the origin of the initiation leg. 

Gate (_phase_b_initiation_no_followthrough):
  previous_phase == "B"
  previous_b_sub_status.startswith("initiation_watch")
  phase_b_initiation_source_anchor_was_run == True

DAG:
  phase = C
  phase_sub_status = "pullback.no_followthrough"
  phase_c_origin_node = f"B.{previous_b_sub_status}"
  origin: B.initiation_watch failed — source ITR anchor run through
C.pullback.after_inducement Archived

Re-entry from a rejected B initiation. The opposing HTF reaction rejected the initiation attempt — treated as an inducement sweep, not a continuation. 

Gate (_phase_b_initiation_rejected):
  previous_phase == "B"
  previous_b_sub_status.startswith("initiation_watch")
  htf_opposing_reaction_rejects == True
  OR initiation anchor lost

DAG:
  phase = C
  phase_sub_status = "pullback.after_inducement"
  phase_c_origin_node = f"B.{previous_b_sub_status}"
  origin: B.initiation_watch rejected — opposing HTF reaction treated as inducement
