Structure Context - SB, ChoCh, P/D Range, Pullback Memory

TL;DR - Structure Context is a Layer 3 lifecycle normalizer over STR / ITR / LTR anchors. Its pro-direction close-through event is named SB / structure_sb (Structure Broken); its counter-direction close-through event is ChoCh / structure_choch. Runtime event names are itrSbUp, itrSbDown, ltrSbUp, and ltrSbDown; runtimeAlias echoes the same canonical name. Structure creates anchor points for State Store history; Orderflow consumes those anchors later under its own distinct orderflow_anchor_sequence lineage. SC05–SC08 are a finer internal layer — iSb and iChoCh events that fire against confirmed ITR/LTR pivot levels rather than the wick-defined range boundaries, adding pivot-level resolution inside the current structural bias.

00 Core Concept

Structure Context detects close-through actions against STR / ITR / LTR-derived structural anchors and owns the market range state needed for Premium / Discount and pullback phase facts.

PE registry / wick range condition met
-> structure lifecycle transition
-> timestamped SC / phase / attempt event
-> current StructureSnapshot
-> Fusion current-truth snapshot
-> Evidence Compiler candidate, if useful to Layer 4

SC events live in Layer 3 because SB / ChoCh needs remembered levels. Layer 2 remains primitive and causal: CE for FVG, PE for pivots. SC is event-shaped, but its detection depends on context memory.

Step 1 · Layer 2 emits primitive PE / CE Layer 2

PE03/PE04 or PE05/PE06 provide confirmed structure points. CE02 can create S/D zones that later confirm pullback.

Step 2 · Structure Context stores lifecycle state Layer 3

State includes warmup registry, wick-defined range, bias, phase, recent levels, last SC, and structure attempts.

Step 3 · Snapshot is projected downstream Fusion / Evidence Compiler

Layer 4 should read compiled context, not rebuild structural meaning from raw PE/CE streams.

What fires looks like an event. What enables the firing requires memory. That is why SC belongs to Layer 3.

01 Lifecycle

The engine has one warmup state and two live phases. neutral is warmup-only and is never re-entered after the first SC fires.

neutral
  warmup registry accepts selected PE tier
  first SC crosses warmup gate

open
  bias is bullish or bearish
  wick range and P/D are live
  extension side updates every bar

pullback_confirmed
  ITR or SD zone confirms peak/trough timing
  wick range stays fixed
  waits for continuation SB or ChoCh

A bias flip transitions directly to open with the opposite bias. Structure does not decide inducement versus real reversal; Layer 4 owns that interpretation.

Range lifecycle

SC01 fires
-> bullish range opens
-> range_low = wick floor
-> range_high = live wick HH while price extends
-> P/D midpoint = midpoint(range_low, range_high)

SC02 fires
-> bearish range opens
-> range_high = wick ceiling
-> range_low = live wick LL while price extends
-> P/D midpoint = midpoint(range_low, range_high)
Without SC there is no range. Without a range there is no P/D zone.

02 Registry vs Range

The most important resolved ambiguity: registry levels and range boundaries are different things.

registry_high / registry_low
  warmup-only selected PE levels
  bootstrap the first SC event
  stop driving SC after warmup

range_high / range_low
  post-warmup wick-defined price boundaries
  drive SB, ChoCh, P/D, and live range projection

pullback_confirmed_by
  timing confirmation only
  values: itr | sd_zone | null
Selected PE tier feeds warmup registry structure.tier 
structure.tier = "itr"
  PE03 / PE04 enter registry
  SC01 / SC02 fire

structure.tier = "ltr"
  PE05 / PE06 enter registry
  SC03 / SC04 fire

STR never enters the structure registry. ITR is the default. LTR is the fallback if ITR proves too fast.

Warmup registry uses most recent level only warmup

In a staircase, the most recently registered selected-tier high or low is the active warmup ceiling/floor. Earlier intermediate levels are internal structure and expire silently when the newest one is broken.

Post-warmup range uses wicks only range

After first SC, newly confirmed PE levels can confirm pullback phase, but they do not overwrite range high or range low. The active price range is wick-defined.

03 SB / ChoCh Detection

Structure actions are close-only. SB / structure_sb means the probe continued through the active structural level in the current movement direction. ChoCh means the probe closed through the opposite structural boundary and flipped structural character. Wick through a level is liquidity behavior, not a structural action.

Warmup detection

structure.tier = itr:
  SC01 itrSbUp     if close > registry_high.price
  SC02 itrSbDown   if close < registry_low.price

structure.tier = ltr:
  SC03 ltrSbUp     if close > registry_high.price
  SC04 ltrSbDown   if close < registry_low.price

Post-warmup detection

bullish context:
  continuation: phase == pullback_confirmed and close > range_high
  ChoCh / flip: close < range_low

bearish context:
  continuation: phase == pullback_confirmed and close < range_low
  ChoCh / flip: close > range_high

Formal close-only condition:

$$ SC_{up} \iff close_t > active\_high \qquad SC_{down} \iff close_t < active\_low $$
Wicks do not fire SC. Time does not decay range boundaries. Only SC01 / SC02 / SC03 / SC04 starts a new leg.
i
Terminology note — SB/ChoCh vs BoS/MSS. Post-warmup SC01–SC04 are conceptually equivalent to what classic SMC literature calls Break of Structure (BoS) and Market Structure Shift (MSS). SB is a pro-bias continuation close-through (= BoS); ChoCh with biasFlip=true is a counter-bias reversal close-through (= MSS). The rename is intentional — "BoS" and "MSS" leaked into Orderflow naming and created ambiguity. Structure now owns only its own confirmed close-through events under canonical names; Orderflow owns its own MSS disruption lineage separately.

Internal iSb / iChoCh — SC05–SC08

SC01–SC04 fire against the wick-defined range boundary. SC05–SC08 fire against individual confirmed ITR/LTR pivot prices (PE03/PE04 for ITR, PE05/PE06 for LTR). They add a finer layer of micro-swing observation inside the current structural bias without replacing or duplicating the macro range events.

Fixed: SC05–SC08 classification is relation-based, not macro-bias-based. iSB fires on continuation relations (HH up, LL down). iChoCh fires on reversal relations (LH up, HL down). These events remain pure observation and do not change bias, phase, or range. 
ITR tier (structure.tier = "itr"):
  SC05  itrISb     close > HH/seed high, or close < LL low
  SC06  itrIChoCh  close > LH high, or close < HL/seed low

LTR tier (structure.tier = "ltr"):
  SC07 / SC08 reserved but disabled in runtime while structure.tier == "ltr"

Key differences from SC01–SC04:

SC01–SC04  fires against wick range_high / range_low
           owns bias and phase transitions
           only fires in pullback_confirmed (continuation) or anytime (ChoCh)

SC05–SC08  fires against PE03/PE04 pivot price (confirmed level)
           pure observation — does NOT change bias, phase, range_high, range_low
           fires post-warmup only
           event type depends on same-side level relation: HH/LH/HL/LL/seed
           co-emit allowed — same bar can return both a SC01–SC04 and a SC05–SC08
i
Why SC05–SC08 exist. SC01–SC04 are the major structure events at the wick-defined range boundary. They are separate from orderflow BoS/MSS. Within a bullish leg there can be multiple ITR highs and lows that price reacts to before the range is broken. SC05–SC08 capture those pivot-level reactions — HH progression that confirms the leg is still running (iSb) or a HL break that suggests weakening momentum (iChoCh) — without conflating them with the major range event. Evidence Compiler and Layer 5 use these for entry qualification.

P/D ratchet reset and anchor recording

P/D range is dynamic current truth. It ratchets on the extension side during open phase, then resets only when an SC lifecycle transition fires. The anchor sequence is historical memory and should record swing points at confirmation / reset moments, not every wick ratchet.

bullish bootstrap:
  first SC up
  -> confirm anchored low from warmup wick floor
  -> open anchor high range
  -> start bullish P/D range

bullish pullback_confirmed:
  -> confirm anchored high from extension extreme
  -> open anchor low candidate range

bullish continuation SB:
  close > range_high
  -> ratchet reset
  -> confirm anchored low from tracked pullback low
  -> open anchor high range

bullish pullback_confirmed -> ChoCh:
  close < range_low
  -> ratchet reset into bearish context
  -> promote anchored high as reversal ceiling
  -> open anchor low range
bearish bootstrap:
  first SC down
  -> confirm anchored high from warmup wick ceiling
  -> open anchor low range
  -> start bearish P/D range

bearish pullback_confirmed:
  -> confirm anchored low from extension extreme
  -> open anchor high candidate range

bearish continuation SB:
  close < range_low
  -> ratchet reset
  -> confirm anchored high from tracked pullback high
  -> open anchor low range

bearish pullback_confirmed -> ChoCh:
  close > range_high
  -> ratchet reset into bullish context
  -> promote anchored low as reversal floor
  -> open anchor high range
i
This recording method naturally tends to emit alternating explicit sides: high anchors are confirmed when an up-leg exhausts or becomes a reversal ceiling; low anchors are confirmed when a down-leg exhausts or becomes a reversal floor. The side is still an owned source fact on each sequence point, not something Orderflow should infer from list position.

04 Pullback Confirmation

Pullback confirmation says the extension likely peaked or troughed. It does not change the P/D price range.

bullish:
  PE03 confirms at extreme -> pullback_confirmed, confirmed_by=itr
  supply zone in premium  -> pullback_confirmed, confirmed_by=sd_zone

bearish:
  PE04 confirms at extreme -> pullback_confirmed, confirmed_by=itr
  demand zone in discount -> pullback_confirmed, confirmed_by=sd_zone
ITR confirmation _confirm_pullback_from_itr

In bullish bias, the selected high PE event (PE03) confirms pullback. In bearish bias, the selected low PE event (PE04) confirms pullback. This is timing context only — the range boundaries do not move.

ITR pivot confirmation can lag 2–3 bars behind the visible price swing peak because PE events fire when the pivot is fully formed (right-hand confirmation bars). The engine confirms pullback at the PE event timestamp, not at the wick extreme bar.

SD zone confirmation _confirm_pullback_from_sd

A bullish pullback can be confirmed by a supply zone whose high is above the P/D midpoint. A bearish pullback can be confirmed by a demand zone whose low is below the midpoint. Confirmation fires on the bar the zone is created.

No resolving phase phase model

pullback_confirmed persists until the next continuation SB or ChoCh. Later same-leg ITR/SD signals are ignored.

Pullback extreme tracking starts at confirmation, not at swing peak _track_pullback_extreme

_pullback_high and _pullback_low are reset to None inside _set_pullback_confirmed. Tracking begins on the confirmation bar and accumulates forward from there.

Consequence: if price peaked at bar T and the ITR pivot confirms at bar T+2, the tracked pullback extreme covers bars T+2 onward — it will miss the T wick if no later bar exceeds it. When a continuation SB fires, the new range ceiling/floor is anchored to the highest/lowest bar seen since confirmation, not since the swing peak.

bearish example:
  bar T     price peaks → visible high ITR
  bar T+2   PE04 fires  → pullback_confirmed, _pullback_high reset to None
  bar T+3   _pullback_high = bar T+3 high   ← tracking starts here
  bar T+5   SB fires → new range_high = _pullback_high (T+3..T+5 max)
            if bar T had the true peak and no bar after T+2 exceeded it,
            new range_high will be lower than the visible swing high

This is intentional: the ratchet anchors the new ceiling to the confirmed pullback window, not to a pre-confirmation wick that may not have been validated by structure.

Structure sequence

Structure Context creates structure-sequence records when P/D lifecycle points become semantically confirmed. The current State Store implementation is the bounded memory inside StructureEngine, exposed through the structure snapshot. This store is the structural SB / ChoCh lineage only. It must not be the long-term source of Orderflow MSS anchors, because a live MSS probe and the later structural ChoCh close-through can otherwise point at the same structural level and look like two independent events to Layer 4.

StructureSequencePoint:
  point_id
  timeframe
  side = high | low
  role = range_origin | extension_extreme | reversal_ceiling | reversal_floor
  price
  timestamp
  confirmed_at
  source_transition = SC01 | SC02 | SC03 | SC04 | pullback_confirmed
  source_event_id

State Store:
  structure_sequence = last 8 confirmed structure points
  recent_structure_sequence_points = alias for snapshot readers
  structure_probe = live +1 unconfirmed probe

Target separation:
  structure_sequence          → structure channel only (SB / ChoCh chronology)
  orderflow_anchor_sequence   → orderflow channel only (orderflow anchor / MSS chronology)
  orderflow_probe             → live probe against protected orderflow anchor
Migration rule: Orderflow may derive its initial anchor-swing store from filtered structure pivots, but once emitted it needs separate IDs, provenance, and lifecycle fields. Layer 4 must be able to tell whether a structural ChoCh is merely the close-confirmation of the same orderflow MSS disruption that opened E.pullback_developing.

ChoCh chronology contract

Structure owns only the confirmed event. It does not decide whether the event confirms a liquidity reclaim, supports Phase D, or occurred inside a valid Layer 4 origin.

last_sc.choch
last_sc.breakDirection
last_sc.eventTimestamp
last_sc.levelPrice
last_sc.biasFlip

Evidence Compiler may qualify:
  last_sc.choch == true
  last_sc.breakDirection is counter to HTF direction
  last_sc.eventTimestamp >= liquidity.reclaimed_at
A counter-direction LTF ChoCh becomes liquidity-reclaim evidence only after Evidence Compiler orders it against a current triggerable liquidity event. Structure must not read Liquidity or Layer 4 phase state.

06 Algorithm

The current engine runs once per completed bar and returns zero or more SC events.

# structureEngine.py - StructureEngine.on_bar()
def on_bar(self, row, pivot_events, ce02_events, bar_result):
    bar_high = row["high"]
    bar_low = row["low"]
    bar_close = row["close"]
    bar_ts = row.name

    # 1. Ingest selected PE tier into recent level memory.
    for pe in pivot_events:
        if pe.event_code == self._pe_high_code:
            self._remember_structure_level(pe, "high")
            self._registry_high = registry_level_from(pe)
        elif pe.event_code == self._pe_low_code:
            self._remember_structure_level(pe, "low")
            self._registry_low = registry_level_from(pe)

    if self._warmup_complete:
        self._maybe_open_pro_attempt_from_break(bar_high, bar_low, bar_ts)

    # 2. Warmup: registry-based SB.
    if not self._warmup_complete:
        self._extend_range_high(bar_high, bar_ts)
        self._extend_range_low(bar_low, bar_ts)
        return maybe_emit(self._structure_break_warmup(bar_close, bar_ts, bar_high, bar_low))

    # 3. Live: wick range + pullback lifecycle.
    if self._phase == "open":
        self._extend_active_side(bar_high, bar_low, bar_ts)
    elif self._phase == "pullback_confirmed":
        self._track_pullback_extreme(bar_high, bar_low, bar_ts)

    self._update_structure_attempt(bar_high, bar_low, bar_close, bar_ts)
    sc = self._structure_break_range(bar_close, bar_ts, bar_high, bar_low)
    if sc:
        return [sc]

    if self._phase == "open":
        self._confirm_pullback_from_itr(pivot_events)
        self._confirm_pullback_from_sd(bar_result.created)
        if pullback_confirmed:
            self._record_structure_point(extension_side, role="extension_extreme")

    return []
# warmup: close against selected registry level
if registry_high and close > registry_high.price:
    warmup_complete = True
    start_bullish(range_low=running_wick_low, bar_high=bar_high)
    record_anchor(low, role="range_origin")
    emit SC01 or SC03

if registry_low and close < registry_low.price:
    warmup_complete = True
    start_bearish(range_high=running_wick_high, bar_low=bar_low)
    record_anchor(high, role="range_origin")
    emit SC02 or SC04
# post-warmup: close against wick-defined range
if bias == "bullish":
    if close < range_low:
        record_or_promote_anchor(high, role="reversal_ceiling")
        start_bearish(prior_high=range_high, bar_low=bar_low)
        emit SC02 or SC04 with biasFlip=True
    elif phase == "pullback_confirmed" and close > range_high:
        record_anchor(low, role="range_origin")
        start_bullish(range_low=pullback_low, bar_high=bar_high)
        emit SC01 or SC03

if bias == "bearish":
    if close > range_high:
        record_or_promote_anchor(low, role="reversal_floor")
        start_bullish(prior_low=range_low, bar_high=bar_high)
        emit SC01 or SC03 with biasFlip=True
    elif phase == "pullback_confirmed" and close < range_low:
        record_anchor(high, role="range_origin")
        start_bearish(range_high=pullback_high, bar_low=bar_low)
        emit SC02 or SC04
# _sb_internal() — fires against PE03/PE04 pivot price, not range boundary
# Called BEFORE _sb_range() so it sees pre-transition structural state.
# Returns at most one internal SC per bar. Does NOT change bias, phase, or range.
def _sb_internal(close, ts):
    if not warmup_complete or bias == "neutral":
        return None

    if lh and close > lh.price:
        if lh.level_relation in {"HH", "seed"}:
            last_isb_level_id = lh.level_id
            emit SC05 itrISb up (choch=False, biasFlip=False)
        else:  # LH high break
            last_ichoch_level_id = lh.level_id
            emit SC06 itrIChoCh up (choch=True, biasFlip=False)

    if ll and close < ll.price:
        if ll.level_relation in {"HL", "seed"}:
            last_ichoch_level_id = ll.level_id
            emit SC06 itrIChoCh down (choch=True, biasFlip=False)
        else:  # LL low break
            last_isb_level_id = ll.level_id
            emit SC05 itrISb down (choch=False, biasFlip=False)

# Dedup state resets when a new structural leg starts (_start_bullish/_start_bearish).
# Warmup pivot is consumed into last_isb_level_id after warmup SC fires,
# preventing the warmup PE from double-firing as an iSb on the first post-warmup bar.
# pro-direction attempt facts
if bias == "bullish":
    trigger = latest_itr_high
    anchor = latest_itr_low
    if bar_high > trigger.price:
        open_attempt(direction="bullish", anchor=anchor)
    if attempt.active and bar_low < anchor.price:
        attempt.status = "failed"
        attempt.failure_reason = "traded_below_itr_low"

if bias == "bearish":
    trigger = latest_itr_low
    anchor = latest_itr_high
    if bar_low < trigger.price:
        open_attempt(direction="bearish", anchor=anchor)
    if attempt.active and bar_high > anchor.price:
        attempt.status = "failed"
        attempt.failure_reason = "traded_above_itr_high"

08 Replayer UI

The replayer renders structure state in two places: chart overlays (P/D range lines) and the right-side Structure panel.

Chart overlays — P/D range lines

Two horizontal lines and an optional midpoint are drawn from the timestamp of the active boundary to the current cursor bar.

range_high line
  starts at: range_high_ts (or phase_start_ts as fallback)
  value: range_high
  color: bearish red (#ef5350) when bias=bearish, teal (#26a69a) when bullish
  style: solid (line_style: 0)

range_low line
  starts at: range_low_ts (or phase_start_ts as fallback)
  value: range_low
  color: same as range_high (bias-driven)
  style: solid (line_style: 0)

P/D midpoint line (optional, show_midpoint: true)
  starts at: phase_start_ts
  value: pd_midpoint
  color: slate (#546e7a)
  style: dashed (line_style: 2)
i
Why range_high can appear to "start late": the line starts at range_high_ts, which is the timestamp of the last bar that extended the high, not the phase start. After pullback_confirmed the range freezes; the line does not extend further right until a SB ratchets it to a new ceiling. A high the eye sees on a pullback bar will not move the line because the engine only extends range_high during open phase.

09 Open Issues · Edge Cases · Next Milestone

Completed · 2026-06-24

Internal pivot events SC05–SC08 (iSB / iChoCh) now use same-side level relation. The old macro-bias read was wrong: it could classify a bullish-context LL break as iChoCh and miss an LH break entirely. The corrected rule stores level_relation on each StructureLevel and classifies internal SC from HH/LH/HL/LL/seed: HH or seed high break → iSB up; LH high break → iChoCh up; HL or seed low break → iChoCh down; LL low break → iSB down.

Archived · Completed

Structural SB / ChoCh ≠ orderflow anchor / MSS. Structure uses STR / ITR / LTR-derived anchors; pro-direction close-through is structure_sb, counter-direction is structure_choch. Orderflow uses its own orderflow_anchor_sequence with distinct provenance and its own MSS disruption events. A close through an ITR boundary can be a structural SB while a later swing-sequence continuation is an independent orderflow anchor event; both may be true but they are not duplicate facts. Structural runtimeAlias now echoes the canonical itrSbUp / itrSbDown names; old itrBosUp-style aliases have been retired.

Edge Case

Cold start. The first SC depends on replay warmup history. The decision is to replay enough recent bars and let the state self-correct chronologically.

Out of Scope

STR SB and sweeps. STR SB is reserved but not implemented. Wick-through sweeps are liquidity context, not structure breaks.

Fallback

ITR too fast. Switch structure.tier to ltr so SC03/SC04 replace SC01/SC02 as the structural driver.

Archived · Moved

Legacy Structural Attempt Facts. This is no longer an active Structure issue. The replacement language belongs to Orderflow / Snapshot Normalizer projection: pressure, disruption, MSS monitor, decision point, protected anchor, and reversal / inducement resolution.
