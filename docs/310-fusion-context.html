Fusion — Current-State Collapse

TL;DR — Fusion reads normalized ContextSnapshot(s) from Snapshot Normalizer and collapses them into a single runtime current truth. In single-TF mode it is an identity pass-through. In dual-TF mode it combines lower and higher snapshots into a DualContextSnapshot that projects HTF objects onto the LTF pane. Fusion does not re-read raw channels, does not qualify evidence, and does not make thesis decisions. That belongs to Evidence Compiler and Layer 4.

00 Core Concept

The three-step pipeline inside Layer 3 after all channel writes are done:

09  Snapshot Normalizer
      reads State Store after Detector writes
      projects one ContextSnapshot per TF
      strictly read-only

10  Fusion
      reads ContextSnapshot(s) from 09
      collapses into runtime current truth
      does not re-read channels

11  Evidence Compiler
      reads the stream of fused snapshots bar by bar
      detects transitions between consecutive snapshots
      emits dated evidence records → evidence candidates → Layer 4

Fusion's place in the pipeline is the handoff boundary between per-channel state and cross-channel current truth. Everything downstream reads the fused snapshot, not raw channel dicts.

Step 1 · Snapshot Normalizer projects per-TF snapshots Node 09

After the Detector finishes writing lifecycle transitions, Snapshot Normalizer reads State Store and projects a clean ContextSnapshot for each active timeframe. Fields include bias, rangeState, pdPosition, nearestSupplyZones, nearestDemandZones, keyLevels, liquidity, orderflow, structureAttempt, and invalidationRefs. All timestamps on individual objects are preserved in full.

Step 2 · Fusion collapses snapshot(s) into current truth Node 10

In single-TF mode Fusion returns the snapshot unchanged. In dual-TF mode Fusion derives cross-TF alignment, narrative, HTF projections, and packs both snapshots into a DualContextSnapshot. It does not re-read structure, SD zones, liquidity, or orderflow engines.

Step 3 · Evidence Compiler reads the fused snapshot stream Node 11

Evidence Compiler sees consecutive fused snapshots, one per bar. It watches for field transitions — rangeState flipping to pullback_confirmed, a zone entering TAPPED, a liquidity pool moving to sweep_confirmed. Those transitions become evidence records with seen_at, confirmed_at, and source object refs.

09 normalizes channels into ContextSnapshot. 10 fuses one or more ContextSnapshots into runtime current truth. 11 compiles transitions and evidence from those fused snapshots.

01 Single vs Dual TF Mode

Mode is determined at the runtime level, not inside Fusion itself.

Single TF
  fused_snapshot = context_snapshot          (identity)
  payload["context_snapshot"] = Fusion.fuse_single(snapshot)

Dual TF
  lower_snapshot  = SnapshotNormalizer.project(LTF cursor + LTF channels)
  higher_snapshot = SnapshotNormalizer.project(HTF cursor + HTF channels)
  fused_snapshot  = Fusion.fuse_dual(lower_snapshot, higher_snapshot,
                      execution_tf=lower_label,
                      reference_tf=higher_label,
                      symbol=symbol)
  payload["context_snapshot"] = fused_snapshot

In dual mode the LTF is always the execution TF. The HTF snapshot is read from what the HTF pipeline has already produced — there is no waiting for an HTF bar close.

No lookahead: only use HTF objects whose timestamps existed before the current LTF bar close.

02 DualContextSnapshot Shape

The output of Fusion.fuse_dual():

mode                      "dual"
symbol                    instrument
currentTimestamp          LTF current bar close timestamp
currentPrice              LTF current bar close price
execution_tf              LTF label  (e.g. "15M")
reference_tf              HTF label  (e.g. "1H")

lower_context_snapshot    full ContextSnapshot for execution TF
higher_context_snapshot   full ContextSnapshot for reference TF
execution_context         alias → lower_context_snapshot
reference_context         alias → higher_context_snapshot

htf_bias                  bullish | bearish | None  (from higher snapshot)
cross_tf_alignment        pullback | continuation | counter_trend | unknown
narrative                 "LTF bullish pullback inside HTF bearish"  | None
execution_pd              premium | discount | midpoint | unknown

projected_htf_levels      HTF keyLevelsAbove + keyLevelsBelow
projected_htf_zones       HTF nearestSupplyZones + nearestDemandZones
confluence_zones          zones present on both TFs  (stub → Evidence Compiler)

cross_tf_alignment logic: if HTF and LTF biases match and rangeState == pullback_confirmed → pullback; same bias, not pullback → continuation; opposite biases → counter_trend.

03 Snapshot Stream & Evidence Compiler

A single fused snapshot answers "what is true right now." It does not record how many bars a condition has held, or what the state was two bars ago.

This does not mean timing is lost. Timestamps on every contained object — levels, zones, anchors, pools — are preserved in full inside the nested ContextSnapshot. What the snapshot does not carry is the transition history between bars.

That history lives in the stream. Evidence Compiler observes the fused snapshot on every bar and records when fields change:

bar N:    rangeState = open,               bias = bullish
bar N+1:  rangeState = pullback_confirmed,  bias = bullish   ← transition
bar N+2:  htf_bias   = bearish,  cross_tf_alignment = counter_trend  ← transition

Each transition becomes a dated evidence record with the source object id, seen_at timestamp, price relation, and orderflow context at the time. Evidence Compiler owns that accumulation. Fusion stays thin.

Do not use a single fused snapshot as a history store. Use the transition stream Evidence Compiler observes across consecutive bars.

04 Algorithm

# core/smc/fusion.py

class Fusion:

    @staticmethod
    def fuse_single(
        context_snapshot: dict
    ) -> dict:
        # Identity pass-through for single-TF mode.
        return context_snapshot
# core/smc/fusion.py

    @staticmethod
    def fuse_dual(
        lower_snapshot: dict,
        higher_snapshot: dict,
        *,
        execution_tf: str,
        reference_tf: str,
        symbol: str,
    ) -> dict:
        htf_bias       = higher_snapshot.get("bias")
        ltf_bias       = lower_snapshot.get("bias")
        ltf_range_state = lower_snapshot.get("rangeState")

        alignment = _cross_tf_alignment(htf_bias, ltf_bias, ltf_range_state)
        narrative = _narrative(htf_bias, ltf_bias, ltf_range_state)

        projected_htf_levels = (
            higher_snapshot.get("keyLevelsAbove", []) +
            higher_snapshot.get("keyLevelsBelow", [])
        )
        projected_htf_zones = (
            higher_snapshot.get("nearestSupplyZones", []) +
            higher_snapshot.get("nearestDemandZones", [])
        )

        return {
            "mode": "dual",
            "symbol": symbol,
            "currentTimestamp": lower_snapshot.get("currentTimestamp"),
            "currentPrice":     lower_snapshot.get("currentPrice"),
            "execution_tf":     execution_tf,
            "reference_tf":     reference_tf,
            "lower_context_snapshot":  lower_snapshot,
            "higher_context_snapshot": higher_snapshot,
            "execution_context":       lower_snapshot,
            "reference_context":       higher_snapshot,
            "htf_bias":             htf_bias,
            "cross_tf_alignment":   alignment,
            "narrative":            narrative,
            "execution_pd":         lower_snapshot.get("pdPosition", "unknown"),
            "projected_htf_levels": projected_htf_levels,
            "projected_htf_zones":  projected_htf_zones,
            "confluence_zones":     _confluence_zones(lower_snapshot, higher_snapshot),
        }
# cross_tf_alignment
def _cross_tf_alignment(htf_bias, ltf_bias, ltf_range_state) -> str:
    if not htf_bias or not ltf_bias:
        return "unknown"
    if htf_bias == ltf_bias:
        if ltf_range_state == "pullback_confirmed":
            return "pullback"
        return "continuation"
    return "counter_trend"

# narrative
def _narrative(htf_bias, ltf_bias, ltf_range_state) -> str | None:
    if not htf_bias or not ltf_bias:
        return None
    if htf_bias == ltf_bias:
        phase = "pullback" if ltf_range_state == "pullback_confirmed" else "continuation"
        return f"LTF {ltf_bias} {phase} inside HTF {htf_bias}"
    return f"LTF {ltf_bias} counter-trend inside HTF {htf_bias}"

# confluence_zones — thin stub
def _confluence_zones(lower_snap, higher_snap) -> list:
    # Evidence Compiler handles cross-TF zone matching.
    return []

05 Open Issues

Out of Scope

Confluence zone matching. _confluence_zones() is a stub. Price-proximity matching across TF zone lists belongs to Evidence Compiler, not Fusion.

Out of Scope

OTE posture. Optimal Trade Entry posture belongs to Layer 5 after Layer 4 defines the active thesis leg. Fusion does not derive entry zones.

Next Milestone

HTF clock guard. Projected HTF levels and zones should carry their source timestamp so Evidence Compiler can reject objects created after the current LTF bar.

Edge Case

HTF not yet warmed up. If higher_context_snapshot.bias is None at session start, cross_tf_alignment returns "unknown" and Evidence Compiler should treat any candidate as blocked until HTF bias resolves.
