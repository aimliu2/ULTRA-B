601 - Layer 6 Trade Analysis — Lightweight Outcome Ledger

TL;DR - Layer 6 V1 should be a small headless outcome recorder, not a broker simulator. It consumes accepted Layer 5 EntryIntent objects, assumes no spread or slippage, classifies win, loss, or timeout, and writes a stable TradeResult row. Skipped Layer 5 decisions can be recorded for audit, but they are not filled trades.
→ 401 Hypothesis Dataflow → 501 Entry Details → 501 Entry Diagram

00 Core Concept

Layer 6 exists so Layer 5 can be tested like a real entry system without immediately binding it to MT5 execution, spread models, partial fills, or order reconciliation. The first version should answer one narrow question: after Layer 5 emits a valid order intent, did that intent win, lose, time out, or get skipped before entry?

This keeps the headless path close to the future bot. Layer 5 emits a broker-neutral EntryIntent; Layer 6 consumes the same shape the bot adapter would later translate into an order. The difference is that V1 fills from replay bars with intentionally simple assumptions.

Layer 6 records outcomes. It does not create entries, alter DAG phase state, or spend budget by itself.

01 Input Contract

Layer 6 receives two intent families from Layer 5. EntryIntent means an order is allowed and can be simulated. SkipIntent means Layer 5 made an explicit no-trade decision that may be useful for later policy analysis.

02 Fill Model

The first fill model should be deliberately simple. It should make Layer 5 measurable without giving false confidence about broker execution. V1 assumes no spread, no slippage, no partial fills, no rejected orders, and one active simulated position per accepted intent.

Discretionary → Algorithm mapping execution simplification

Discretionary idea: if the entry quality is gone, do not chase. If Layer 5 still allows the trade, test the clean thesis without adding broker noise yet.

Algorithm mapping: fill accepted MP entries at the intent price, then scan future bars until SL, TP, or timeout fires.

FillModelV1:
  fill_policy: no_spread_no_slippage
  entry_fill: intent.entry_price
  active_until:
    stop_loss touched
    target_price touched
    timeout reached
    dataset ends
This model is a baseline. Spread, slippage, broker minimum stop distance, partial fill, and MT5 reconciliation are later adapters, not Layer 6 V1 requirements.

03 Outcome Rules

Outcome rules must be deterministic so policy comparisons are repeatable. The ambiguous case is a bar that touches both SL and TP. V1 should use loss_first because intrabar path is unknown and optimism would overstate edge.

Touch rules bar replay 
Long:
  if bar.low  <= stop_loss:    loss
  if bar.high >= target_price: win

Short:
  if bar.high >= stop_loss:    loss
  if bar.low  <= target_price: win

Same bar touches both:
  same_bar_sl_tp_policy = loss_first
Timeout rules unfinished thesis 
Timeout:
  if neither SL nor TP touched within max_hold_bars:
    outcome = timeout
    exit_time = timeout_bar.timestamp
    exit_price = timeout_bar.close
    r_result = mark_to_market_r(exit_price)

05 Budget / Stale Interaction

Layer 5 owns budget and stale ledgers. Layer 6 should record the consequences, not become the owner of those ledgers. This keeps the bot path straightforward: broker execution reports can later update the same result fields without moving permission logic into analysis.

Budget rule Layer 5 remains owner 
If EntryIntent is accepted:
  budget_spent = true       # placed or filled order, depending adapter policy
  Layer 6 records final outcome

If SkipIntent is emitted:
  budget_spent = false
  outcome = skipped
  skip_reason preserved

If skip_reason == late_entry_risk_too_wide:
  stale_marked = true
  same phase episode / evidence / trigger is blocked by Layer 5

06 Metrics

The first reports should be diagnostic, not optimized. The goal is to see whether the entry contract behaves sanely: D.medium should not chase, skipped late entries should preserve budget, and timeouts should reveal unclear geometry or weak target assumptions. 

Minimum report:
  filled_trades
  skipped_intents
  wins
  losses
  timeouts
  win_rate
  avg_r
  expectancy_r
  timeout_rate
  stale_skip_count
  by_phase
  by_policy
  by_evidence_kind
  by_trigger_kind
Fixed 2R/2.5R is the baseline. Daily ATR geometry belongs in a later Layer 6 experiment after the result ledger is stable.

07 Algorithm

The algorithm is a small reducer over Layer 5 intents and replay bars. It should be reusable by a headless backtest, replay UI, and eventually the bot's dry-run mode. 

# Layer 6 outer loop
def on_intent(intent, cursor):
    if intent.kind == "skip":
        return TradeResult.skipped(
            intent=intent,
            stale_marked=intent.stale_marked,
        )

    trade = ActiveTrade.from_entry_intent(intent)

    while cursor.has_next_bar():
        bar = cursor.next_bar()
        result = advance_trade(trade, bar)
        if result is not None:
            return result

    return TradeResult.timeout_or_open_end(trade, cursor.last_bar)
# Deterministic touch classifier
def advance_trade(trade, bar):
    if trade.direction == "long":
        hit_sl = bar.low <= trade.stop_loss
        hit_tp = bar.high >= trade.target_price
    else:
        hit_sl = bar.high >= trade.stop_loss
        hit_tp = bar.low <= trade.target_price

    if hit_sl and hit_tp:
        return close_trade(trade, bar, outcome="loss")  # loss_first
    if hit_sl:
        return close_trade(trade, bar, outcome="loss")
    if hit_tp:
        return close_trade(trade, bar, outcome="win")
    if trade.bars_held >= trade.max_hold_bars:
        return close_trade(trade, bar, outcome="timeout", exit_price=bar.close)
    return None
def close_trade(trade, bar, outcome, exit_price=None):
    exit_price = exit_price or (
        trade.stop_loss if outcome == "loss" else trade.target_price
    )
    r_result = signed_r(
        direction=trade.direction,
        entry=trade.entry_price,
        exit=exit_price,
        risk=abs(trade.entry_price - trade.stop_loss),
    )
    return TradeResult(
        intent_id=trade.intent_id,
        outcome=outcome,
        exit_time=bar.timestamp,
        exit_price=exit_price,
        r_result=r_result,
        bars_held=trade.bars_held,
    )

08 Config

The runtime does not yet expose a Layer 6 config block. The following shape is the proposed implementation contract. 

replay:
  trade_analysis:
    enabled: true
    fill_model: no_spread_no_slippage
    same_bar_sl_tp_policy: loss_first
    timeout:
      max_hold_bars: 32
      exit_price: close
    record_skips: true
    result_log:
      enabled: true
      format: csv
      path: trade_results.csv
class TradeAnalysisConfig:
    def __init__(self, raw):
        cfg = raw.get("replay", {}).get("trade_analysis", {})
        self.enabled = bool(cfg.get("enabled", False))
        self.fill_model = cfg.get("fill_model", "no_spread_no_slippage")
        self.same_bar_sl_tp_policy = cfg.get("same_bar_sl_tp_policy", "loss_first")
        self.max_hold_bars = int(cfg.get("timeout", {}).get("max_hold_bars", 32))
        self.timeout_exit_price = cfg.get("timeout", {}).get("exit_price", "close")
        self.record_skips = bool(cfg.get("record_skips", True))

09 Open Issues

Next Milestone

Implement Layer 5 first enough to emit intents. Layer 6 depends on a stable EntryIntent / SkipIntent stream.

Open Issue

Same-bar path is unknown. V1 uses loss_first. Later testing can compare open/high/low/close path assumptions or lower-timeframe replay.

Out of Scope

Broker realism. Spread, slippage, min stop distance, order rejection, and partial fill are deliberately deferred to the bot adapter/backtest realism phase.

Out of Scope

Daily ATR geometry. ATR-shaped stops/targets should be tested after fixed 2R/2.5R produces a reliable baseline ledger.

Edge Case

Dataset end before timeout. Mark as timeout with an audit flag or separate open_end later if this becomes common in replay windows.

Fallback

If metrics look too optimistic. Keep the same Layer 5 intents and rerun Layer 6 with spread/slippage and stricter same-bar policy variants.
