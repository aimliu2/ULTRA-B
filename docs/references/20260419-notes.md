### Fix 1 ‚Äî `_hour_in_range` extracted from nested function to module level

**Problem**
`_in_range` was defined *inside* `_session_allowed` on every call:

```python
def _session_allowed(...):
    if allowed_hours is not None:
        def _in_range(h, start, end):   # re-defined every call
            ...
        return any(_in_range(...) for ...)
```

`_session_allowed` is called 3√ó per bar (once per hypothesis). Python re-executes
the `def` statement each time ‚Äî ~150k+ unnecessary function definitions over a full
backtest.

**Fix**
Extracted to module-level `_hour_in_range(h, start, end)` with wrap-around support.
Both `session_label_for` and `_session_allowed` now call the shared function.

```python
def _hour_in_range(h: int, start: int, end: int) -> bool:
    if start <= end:       # normal:    [7, 21]
        return start <= h < end
    return h >= start or h < end  # overnight: [21, 5]
```

### Files changed
- `backtester/strategy.py` ‚Äî `_hour_in_range` added at module level; `_session_allowed`
  and `session_label_for` updated to call it; nested `_in_range` removed

### Fix 3 ‚Äî Dead `sl_offset` / `tp_offset` fields removed from all candidates

**Problem**
The A1 block computed SL/TP with a fake `entry_price=0.0` placeholder and stored
the results as `sl_offset` / `tp_offset` in the candidate dict:

```python
sl, tp = a1_sl_tp(direction, entry_price=0.0, ...)  # fake price
fired.append({
    'sl_offset': sl,    # never read
    'tp_offset': tp,    # never read
    '_ema20': ...,
    '_st_line': ...,
})
```

`resolve_sl_tp()` ‚Äî which runs at fill time with the real entry price ‚Äî never reads
`sl_offset` or `tp_offset`. It recalculates everything from `_ema20`, `_st_line`,
`_a1_sl_mode`, `_a1_rr`. The pre-computation was pure wasted work.

A2 and B already carried `'sl_offset': None, 'tp_offset': None` (placeholder Nones
going nowhere), confirming the fields were unused across all three hypotheses.

**Fix**
- Removed the `a1_sl_tp(entry_price=0.0)` pre-computation call from A1 block
- Removed `sl_offset` and `tp_offset` from A1, A2, and B candidate dicts
- Comment on remaining `_` fields updated: "SL inputs ‚Äî resolve_sl_tp uses these
  at fill time with real entry price"

### Files changed
- `backtester/strategy.py` ‚Äî dead pre-computation and fields removed from all three
  hypothesis candidate dicts in `evaluate_hypotheses()`
