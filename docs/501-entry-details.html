501 - Layer 5 Entry — MP Entry, Budgets, iChoCh

TL;DR — Layer 5 V1 is an MP-only entry permission layer over the DAG snapshot. Phase D is the first fully implemented entry phase. D.watch has three trigger paths: SA — no HTF zone context, fresh iChoCh fires during D.watch, SL = phase_d_shadow.watch_range_extreme + buffer; Path B — HTF SD zone context, direction-correct LTF ITR pivot arms the zone probe, then fresh iChoCh confirms entry, SL = HTF zone distal/invalidation edge + buffer; C2 — C.pullback from D.watch_mss, transition-bar entry, SL = D.watch extreme. Phase E and Phase X are no-trade by default. Risk above 25 pips (EURUSD) marks the opportunity stale and saves budget. Symbol geometry (sl_band_pips, sl_buffer_pips) is per-symbol from config.yaml asset_geometry — unknown symbol returns None, no silent fallback. TP = pd_midpoint (50% HTF PD range); RR ≥ 1.75 to take.

00 DAG / Layer 5 Boundary

The DAG (Layer 4) owns phase state transitions. It uses only macro structural events (last_sc, SC01–SC04) because these fire reliably against the wick-defined range boundary and are structurally unambiguous. iChoCh (SC05/SC06, last_isc) fires against confirmed internal ITR pivot levels — earlier and finer, but not a DAG gate.

Layer 5 reads the snapshot after the DAG has committed phase. At D.watch, the DAG is holding — the bounce started but no counter MSS has fired yet to force C.pullback. Layer 5 reads the same EC candidate facts to decide if iChoCh or a macro SB is present and whether that is enough confirmation to enter. 

DAG (Layer 4) uses:
  last_sc  (SC01/SC02/SC03/SC04)  — macro ChoCh and SB, wick range boundary
  ← all DAG phase transitions gated on last_sc only

Layer 5 uses additionally:
  last_isc (SC05/SC06)            — internal iChoCh, confirmed ITR pivot break
  ltf_counter_choch_seen          — EC debug fact (from ltf_counter_choch candidate)
  ltf_counter_isb_seen            — proposed EC debug fact for same-direction internal iSB continuation
  ltf_counter_sb_seen             — EC debug fact (from same candidate)
  counter_entry_evidence          — Layer 5 evidence object resolved from EC/context facts

Layer 4 and Layer 5 read different keys from the same EC candidates.
Layer 5 may also resolve raw zone/liquidity IDs from the context snapshot for SL geometry.
The boundary is not about signal strength — it is about responsibility. The DAG holds structural commitment memory; Layer 5 holds entry permission state. iChoCh is an entry confirmation signal, not a phase-advance signal.
Regime tags analysis-only Layer 5 context

Regime tags are factual labels derived from the committed snapshot and Layer 5 decision. They are written to the backtest CSVs for filtering and later policy tightening, but they do not change DAG phase state and do not block or allow entries yet.

07 Epoch Budget

Layer 5 budgets are scoped to the active HTF P/D epoch. In current runtime language, the durable discriminator is htf_pd_epoch_id; it changes when the HTF P/D level lifecycle changes and survives ordinary phase oscillation inside the same thesis. Use epoch for this HTF lifecycle. Use phase episode only for a local visit to D.watch, C.pullback, B.watch, or A.watch inside that epoch.

08 MP Risk Geometry

V1 supports only MP entry. The entry protocol does not define limit ladders, controlled chase, partial fills, or alternate execution modes yet. It reads the current snapshot, validates phase-specific evidence and trigger order, computes SL from the selected evidence invalidation boundary, and accepts only trades whose execution quality is still inside the configured asset risk band. 2R is the general target profile and 2.5R is reserved for wide HTF runway or outside-range invalidation; later Layer 6 tests may replace fixed RR with a Daily ATR geometry shape per asset.

MP order geometry V1 execution mode 
SL geometry:
  stop_loss  = evidence extreme + sl_buffer_pips (fixed: 2 pips EURUSD)
  sl_pips    = abs(entry_price - stop_loss) / pip_size
  valid band: 15 to 25 pips
    sl_pips > 25: skip — late_entry_risk_too_wide, save budget, block episode
    sl_pips < 15: widen SL outward to the configured minimum floor

TP geometry:
  take_profit = higher_structure.pd_midpoint   ← 50% of HTF PD range
                already in snapshot: (range_high + range_low) / 2
  tp_pips     = abs(entry_price - take_profit) / pip_size
  rr          = tp_pips / sl_pips

RR gate:
  rr < 1.75: skip — pd_midpoint too close, not worth the risk
  rr >= 1.75: accept — TP stays at pd_midpoint, no upper cap
              rr > 2.5 means wider runway, still a better trade

10 Algorithm

Layer 5 reads the DAG snapshot bar-by-bar. It tracks its own MP attempts, resolved outcomes, phase episode status, and epoch budget separately from the DAG. The DAG snapshot is the clock; Layer 5 applies permission rules on each bar.

Open Issue

Budget ownership and self-relocation reset. EntryBudgetLedger and _d_watch_budget_spent are Layer 5 in-memory state — not persisted across bot restarts. After any restart, budget resets to zero regardless of what trades were accepted in the prior session. Shadow Thesis self-relocation (terrain relocation after restart) may also produce a new phase_episode_id, allowing Layer 5 to re-enter an episode that was already spent. Resolution options: (1) persist the budget ledger alongside Shadow Thesis journal (keyed by htf_pd_epoch_id + direction); (2) accept the gap for now and rely on the epoch discriminator to bound re-entry risk. Deferred until entry logic is accurate enough to run the null case.

Open Issue

A.watch oscillation in ranging markets burns entry budget. When the market drifts sideways inside a B episode, A.watch ↔ A.watch_weaken flips repeatedly — any micro counter MSS triggers weaken; any micro pro SC triggers recovery. Each A.watch bar looks like a valid entry. Layer 5 needs: (1) a hard entry cap per phase_episode_id (2–3 max); (2) after SL hit, require higher-quality confirmation (pro extreme advance, not just pro SC) before re-entry; (3) an episode staleness guard — if A.watch has been active N bars with no new pro-side extreme, treat as ranging and block new entries until a full reset (C.pullback or E.seeking). The DAG-side fix (pro extreme advance gate on A.watch_weaken → A.watch recovery) reduces the oscillation frequency but does not eliminate the need for Layer 5 budget control.
