501 - Layer 5 Entry — MP Entry, Budgets, iChoCh

TL;DR — Layer 5 V1 is an MP-only entry permission layer over the DAG snapshot. Phase D has one active D.watch hold-entry path: D.watch_pathA, internal counter iChoCh → internal counter iSB (EX.entry — both breaks in counter-HTF direction), SL = phase_d_shadow.watch_range_extreme + buffer with the configured minimum floor, TP = pd_midpoint. HTF S/D zone context is an analysis tag, not a separate trigger or SL anchor. D.watch → C.pullback MSS is a handoff, not a D trade. Phase B is implemented with B.watch_pathA (counter iChoCh → retracement any form, not gated → pro iChoCh) and B.watch_pathB (one-shot B.watch → A.watch major pro structure transition after step 1 was armed). Phase B uses phase_b_shadow_commitment_extreme_level (locked at C→B) for SL, the Phase A objective-progress level capped at 2.5R for TP, and a per-B-episode consumption guard. Phase A has two entry slots: A.watch uses OTE.entry (pro-HTF, SL = phase_a_shadow_commitment_extreme_level — Shadow Thesis snap, re-snapped at each A.watch open); A.watch_weaken uses EX.entry (pro iChoCh → pro iSB, SL = phase_a_shadow_watch_range_extreme + buffer, no clamp, 15–25 pip band; TP = 90% objective capped at 2.5R) — after EX.entry: Layer 5 treats Phase A as exhausted (budget forced to 3); DAG defers X.thesis_over via phase_a_thesis_matured (price progress). Phase C mirrors Phase A in counter-HTF direction (OTE.entry at C.pullback, EX.entry at C.pullback_weaken, independent budget, no X.thesis_over after EX) — note only, implementation pending. Phase E and Phase X are no-trade by default. SL_too_wide skip is non-stale (stale=False); trigger re-evaluates until age-out; no budget consumed. Symbol geometry (sl_band_pips, sl_buffer_pips) is per-symbol from config.yaml asset_geometry — unknown symbol returns None, no silent fallback. RR ≥ 1.75 to take; runway_too_short may retry the same trigger within the trigger-age cap.

00 DAG / Layer 5 Boundary

The DAG (Layer 4) owns phase state transitions. It uses only macro structural events (last_sc, SC01–SC04) because these fire reliably against the wick-defined range boundary and are structurally unambiguous. iChoCh (SC05/SC06, last_isc) fires against confirmed internal ITR pivot levels — earlier and finer, but not a DAG gate.

Layer 5 reads the snapshot after the DAG has committed phase. At D.watch, the DAG is holding — the bounce started but no counter orderflow MSS has fired yet to force C.pullback. Layer 5 reads EC internal-structure sequence facts to decide if the D.watch exhaustion entry is allowed. 

DAG (Layer 4) uses:
  last_sc  (SC01/SC02/SC03/SC04)  — macro ChoCh and SB, wick range boundary
  ← all DAG phase transitions gated on last_sc only

Layer 5 uses additionally:
  internal_structure_sequence      — internal iChoCh/iSB chronology
  ltf_counter_ichoch_*             — EC debug facts for internal counter iChoCh
  ltf_pro_ichoch_*                 — EC debug facts for internal pro iChoCh
  ltf_counter_sequence_*           — EC debug facts for counter iChoCh → counter iSB
  ltf_counter_sb_seen              — major counter SB quality context, not a D trigger
  counter_entry_evidence          — Layer 5 evidence object resolved from EC/context facts

Layer 4 and Layer 5 read different keys from the same EC candidates.
Layer 5 may also resolve raw zone/liquidity IDs from the context snapshot for SL geometry.
The boundary is not about signal strength — it is about responsibility. The DAG holds structural commitment memory; Layer 5 holds entry permission state. iChoCh is an entry confirmation signal, not a phase-advance signal.
Regular entry patterns Layer 5 vocabulary 
OTE.entry:
  A reusable Layer 5 regular pattern, not a Phase B-only special case.

  Shape:
    1. phase opens with a fixed commitment / invalidation extreme
    2. counter-side internal structure appears
       minimum: SC06 counter iChoCh
    3. retracement (any form, not gated explicitly)
       accepts: ITR pivot, sideways chop, iSB chain in counter direction, or immediate reversal
       temporal ordering enforced by step 4 (pro signal must be after step 2)
    4. pro-side resumption signal fires after step 2
       minimum: SC06 pro iChoCh while the phase still holds
       stronger: major pro structure transition may move the DAG to A; counter MSS may move the DAG to C
    5. entry uses the phase's commitment extreme as SL anchor

  Budget ownership:
    if step 1 was not observable before the phase transition,
      skip this phase and let the next phase own any trade
    if step 1 was observable and the transition is step 4,
      this is still the prior phase's OTE.entry and spends prior-phase budget

  Current concrete instances:
    B.watch_pathA / B.watch_pathB — commitment_extreme_level (locked at C→B).
    A.watch_pathA — commitment_extreme_level (Shadow Thesis snap, re-snapped at each A.watch open).
    C.pullback OTE.entry (counter-HTF mirror) — C-specific commitment extreme; pending.
    Any new variant must define its own commitment extreme, retracement evidence, and budget owner.

EX.entry:
  A 2-step structural confirmation pattern. Named for the exhaustion sequence —
  two consecutive same-direction internal structure breaks confirm a directional move
  before any opposing candle sequence appears.

  Direction: the two-step sequence direction matches the phase's trade direction.
    Phase D (counter-HTF trade): counter SC06 iChoCh → counter SC05 iSB → enter counter-HTF.
    Phase A (pro-HTF trade):     pro SC06 iChoCh   → pro SC05 iSB   → enter pro-HTF.
  Same 2-step shape applied symmetrically in each phase's trade direction.

  Phase D concrete shape (D.watch_pathA):
    1. D.watch holds; watch_range_extreme tracks the counter-side running extreme
    2. Counter SC06 iChoCh fires (counter internal ChoCh — sequence starts)
    3. Counter SC05 iSB fires after the iChoCh (same counter-HTF direction)
    4. entry at iSB bar close
       SL = watch_range_extreme ± buffer; 15–25 pip band (EURUSD)
       HTF zone context is recorded as a regime tag only; it is not an SL anchor
    Budget: spends Phase D budget; tagged D.watch_pathA.

  Phase A concrete shape (A.watch_weaken_ex):
    1. A.watch_weaken is active; pro_extreme_at_weaken is set
    2. Pro SC06 iChoCh fires after weaken_at (resumption starts)
    3. Pro SC05 iSB fires after the iChoCh (same pro-HTF direction)
    4. entry at iSB bar close
       SL = phase_a_shadow_watch_range_extreme ± buffer; 15–25 pip band (no clamp)
       TP = Phase A objective-progress level capped at 2.5R
    Budget: spends Phase A retry slot; after EX.entry: Layer 5 forces Phase A budget to 3 — DAG defers X.thesis_over via phase_a_thesis_matured.

  Key distinction from OTE.entry:
    OTE: counter iChoCh → retracement (any form) → pro signal (direction flips at final step).
    EX:  two consecutive same-direction iChoCh + iSB (no retracement, no direction flip).

Layer 5 can eventually define C first-entry and re-entry rules around iChoCh, but the exact C rule remains a draft until the phase-specific iChoCh spec is completed.

C.pullback_weaken is not automatically invalid. It marks a pro-HTF bounce attempt inside the counter pullback. Layer 5 should treat it as context, then wait for the C-specific iChoCh rule before spending a C ticket.
Edge case — boundary skip is intentional. When the LTF P/D range is so compressed or fast-moving that the current phase did not arm its own entry setup, Layer 5 should skip that phase instead of forcing a boundary trade. An early skip is correct behavior: it prevents a FOMO entry and leaves the next phase quota available. Example: D.watch → C.pullback MSS hands ownership to Phase C; Phase D does not spend just because the transition bar exists.

04 Phase B Entry

Phase B is the pro-HTF return attempt after C has reached the required pullback depth. B.watch_pathA and B.watch_pathB implement OTE.entry with one trade attempt per B.watch episode after an accepted entry. SL_too_wide is a non-stale geometry reject and does not consume the episode. Trade direction is pro-HTF — the same as direction, opposite of Phase D's counter-HTF convention.

08 MP Risk Geometry

V1 supports only MP entry. The entry protocol does not define limit ladders, controlled chase, partial fills, or alternate execution modes yet. It reads the current snapshot, validates phase-specific evidence and trigger order, computes SL from the selected evidence invalidation boundary, and accepts only trades whose execution quality is still inside the configured asset risk band. Phase D targets pd_midpoint. Phase B targets the configured Phase A objective-progress level, capped at 2.5R. Later Layer 6 tests may replace these target shapes with a Daily ATR geometry shape per asset.

MP order geometry V1 execution mode 
SL geometry:
  stop_loss  = evidence extreme + sl_buffer_pips (fixed: 2 pips EURUSD)
  sl_pips    = abs(entry_price - stop_loss) / pip_size
  valid band: 15 to 25 pips
    sl_pips > 25: skip — SL_too_wide (stale=False); no budget consumed; trigger re-evaluates until age-out
    sl_pips < 15: widen SL outward to the configured minimum floor

D TP geometry:
  take_profit = higher_structure.pd_midpoint   ← 50% of HTF PD range

B TP geometry:
  progress_level = pd_low + threshold * span       (long)
                 = pd_high - threshold * span      (short)
  threshold = replay.hypothesis.phase_a.objective_progress_threshold
  tp_pips = min(abs(progress_level - entry_price), 2.5 * sl_pips)

RR gate:
  rr < 1.75: skip — runway_too_short, retryable while trigger age is valid
  rr >= 1.75: accept

Snapshot fields used (higher_structure):
  range_high    float   ← top of active HTF PD range
  range_low     float   ← bottom of active HTF PD range
  pd_midpoint   float   ← (range_high + range_low) / 2, confirmed live in snapshot

Phase D TP comes from pd_midpoint; Phase B TP comes from the Phase A objective-progress level capped at 2.5R. The RR is derived, then gated: if rr < 1.75, emit runway_too_short; if rr ≥ 1.75, accept.
