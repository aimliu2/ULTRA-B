501 - Layer 5 Entry — MP Entry, Budgets, iChoCh

TL;DR — Layer 5 V1 is an MP-only entry permission layer over the DAG snapshot. Phase D has one active D.watch hold-entry path: D.watch_pathA, internal counter iChoCh → internal counter iSB (EX.entry — both breaks in counter-HTF direction), SL = phase_d_shadow.watch_range_extreme + buffer with the configured minimum floor, TP = pd_midpoint. HTF S/D zone context is an analysis tag, not a separate trigger or SL anchor. D.watch → C.pullback MSS is a handoff, not a D trade. Phase B is implemented with B.watch_pathA (counter iChoCh → retracement any form, not gated → pro iChoCh) and B.watch_pathB (one-shot B.watch → A.watch major pro structure transition after step 1 was armed). Phase B uses the B commitment extreme for SL, the Phase A objective-progress level capped at 2.5R for TP, and a per-B-episode consumption guard. Phase E and Phase X are no-trade by default. Risk above 25 pips (EURUSD) marks the opportunity stale and saves budget. Symbol geometry (sl_band_pips, sl_buffer_pips) is per-symbol from config.yaml asset_geometry — unknown symbol returns None, no silent fallback. RR ≥ 1.75 to take; runway_too_short may retry the same trigger within the trigger-age cap.

00 DAG / Layer 5 Boundary

The DAG (Layer 4) owns phase state transitions. It uses only macro structural events (last_sc, SC01–SC04) because these fire reliably against the wick-defined range boundary and are structurally unambiguous. iChoCh (SC05/SC06, last_isc) fires against confirmed internal ITR pivot levels — earlier and finer, but not a DAG gate.

Layer 5 reads the snapshot after the DAG has committed phase. At D.watch, the DAG is holding — the bounce started but no counter orderflow MSS has fired yet to force C.pullback. Layer 5 reads EC internal-structure sequence facts to decide if the D.watch exhaustion entry is allowed. 

DAG (Layer 4) uses:
  last_sc  (SC01/SC02/SC03/SC04)  — macro ChoCh and SB, wick range boundary
  ← all DAG phase transitions gated on last_sc only

Layer 5 uses additionally:
  internal_structure_sequence      — internal iChoCh/iSB chronology
  ltf_counter_ichoch_*             — EC debug facts for internal counter iChoCh
  ltf_pro_ichoch_*                 — EC debug facts for internal pro iChoCh
  ltf_counter_sequence_*           — EC debug facts for counter iChoCh → counter iSB
  ltf_counter_sb_seen              — major counter SB quality context, not a D trigger
  counter_entry_evidence          — Layer 5 evidence object resolved from EC/context facts

Layer 4 and Layer 5 read different keys from the same EC candidates.
Layer 5 may also resolve raw zone/liquidity IDs from the context snapshot for SL geometry.
The boundary is not about signal strength — it is about responsibility. The DAG holds structural commitment memory; Layer 5 holds entry permission state. iChoCh is an entry confirmation signal, not a phase-advance signal.
Regular entry patterns Layer 5 vocabulary 
OTE.entry:
  A reusable Layer 5 regular pattern, not a Phase B-only special case.

  Shape:
    1. phase opens with a fixed commitment / invalidation extreme
    2. counter-side internal structure appears
       minimum: SC06 counter iChoCh
    3. retracement (any form, not gated explicitly)
       accepts: ITR pivot, sideways chop, iSB chain in counter direction, or immediate reversal
       temporal ordering enforced by step 4 (pro signal must be after step 2)
    4. pro-side resumption signal fires after step 2
       minimum: SC06 pro iChoCh while the phase still holds
       stronger: major pro structure transition may move the DAG to A; counter MSS may move the DAG to C
    5. entry uses the phase's commitment extreme as SL anchor

  Budget ownership:
    if step 1 was not observable before the phase transition,
      skip this phase and let the next phase own any trade
    if step 1 was observable and the transition is step 4,
      this is still the prior phase's OTE.entry and spends prior-phase budget

  Current concrete instance:
    B.watch_pathA / B.watch_pathB use commitment_extreme_level.
    Future A/C variants should reuse this pattern only after defining their
    phase-specific commitment extreme, retracement evidence, and budget owner.

EX.entry:
  Phase D's entry pattern (D.watch_pathA). Named for the exhaustion sequence
  — the counter move stalls via iChoCh→iSB before the pro resumption.

  Shape:
    1. D.watch holds; watch_range_extreme tracks the counter-side running extreme
    2. Counter SC06 iChoCh fires (counter internal ChoCh — sequence starts)
    3. Counter SC05 iSB fires after the iChoCh (same counter-HTF direction)
       (second structural break in counter direction confirms exhaustion of the pro move)
    4. entry at iSB bar close
       SL = watch_range_extreme ± buffer (always)
       HTF zone context is recorded as a regime tag only; it is not an SL anchor

  Budget ownership:
    spends Phase D budget; tagged D.watch_pathA.

  Key distinction from OTE.entry:
    OTE: counter iChoCh → retracement (any form) → pro iChoCh (direction flips at step 4).
    EX: counter iChoCh → counter iSB (same counter-HTF direction, no retracement step).
    EX is the Phase D signal; OTE is the Phase B signal.

03 Phase C Entry

Phase C is the confirmed counter-pullback phase. The DAG can enter C from D.watch via fresh counter MSS or directly from E.pullback_developing when depth reaches the fast-path threshold. Layer 5 can eventually define C first-entry and re-entry rules around iChoCh, but the exact C rule remains a draft until the phase-specific iChoCh spec is completed.

C.pullback_weaken is not automatically invalid. It marks a pro-HTF bounce attempt inside the counter pullback. Layer 5 should treat it as context, then wait for the C-specific iChoCh rule before spending a C ticket.
Edge case — boundary skip is intentional. When the LTF P/D range is so compressed or fast-moving that the current phase did not arm its own entry setup, Layer 5 should skip that phase instead of forcing a boundary trade. An early skip is correct behavior: it prevents a FOMO entry and leaves the next phase quota available. Example: D.watch → C.pullback MSS hands ownership to Phase C; Phase D does not spend just because the transition bar exists.

04 Phase B Entry

Phase B is the pro-HTF return attempt after C has reached the required pullback depth. B.watch_pathA and B.watch_pathB implement OTE.entry with one trade attempt per B.watch episode after an accepted entry or stale geometry skip. Trade direction is pro-HTF — the same as direction, opposite of Phase D's counter-HTF convention.

  if weaken is active and pro-side extreme has not advanced:

Open Issue

Budget ownership and self-relocation reset. EntryBudgetLedger and _d_watch_budget_spent are Layer 5 in-memory state — not persisted across bot restarts. Open Issue

Each A.watch bar looks like a valid entry. Layer 5 needs: (1) a hard entry cap per phase_episode_id (2–3 max); (2) after SL hit, require higher-quality confirmation (pro extreme advance, not just pro SC) before re-entry; (3) an episode staleness guard — if A.watch has been active N bars with no new pro-side extreme, treat as ranging and block new entries until a full reset (C.pullback or E.seeking). The DAG-side fix (pro extreme advance gate on A.watch_weaken → A.watch recovery) reduces the oscillation frequency but does not eliminate the need for Layer 5 budget control.
