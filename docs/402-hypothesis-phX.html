Hypothesis Phase X & none — thesis.over

TL;DR — phase="none" has been eliminated. All silence states now emit phase="X" with a phase_sub_status: X.warm_up (no HTF structure), X.no_direction (neutral bias), X.none (unclassified gap), X.thesis_over (post-cycle terminal). The _none() method and none_sub_status field are deleted. X.thesis_over is the only sub-status with entry_policy="skip" — it fires after _phase_a_finale() confirms the configured Phase A objective-progress maturity threshold was reached.
→ Phase A — A.watch → DAG diagram

00 Overview

The hypothesis DAG now uses one phase value for "not actively trading": phase="X". The old literal "none" is gone; intent is carried by phase_sub_status. Pre-cycle silence, neutral bias, unclassified fallthrough, and post-cycle terminal state all share the same surface behaviour — direction is "none", status is "watching" — but remain distinct by sub-status and debug facts.

# Pre-cycle / unclassified silence
	warmup / no HTF structure      →  X.warm_up
	HTF bias neutral               →  X.no_direction
	unclassified fallthrough       →  X.none

	# Active cycle
	E → D → C → B → A

	# Post-cycle terminal
	A.watch + _phase_a_finale()    →  X  (X.thesis_over)
  entry_policy: "skip"
  required_evidence: ["phase_a_thesis_matured"]
  invalidation: new Phase E resets the P/D range episode
X.warm_up and X.no_direction are expected wait states. X.none is different: it means the HTF bias is directional but no DAG block claimed the bar. It now emits compact blocked-transition debug so replay can show why the fallthrough happened.

01 Phase X — Current Emit Sites

_none() is deleted. none_sub_status field removed from Hypothesis. All 4 former _none() call sites now call _phase_x(phase_sub_status=...). The Phase literal no longer contains "none".

phase_sub_status Trigger condition entry_policy
X.warm_up HTF snapshot absent, no closed HTF bar seen yet, or classifier state not ready. Maps from: warmup_waiting_for_first_closed_htf, waiting_for_htf_structure, waiting_for_hypothesis_state. "wait"
X.no_direction HTF structure present but bias == "neutral" or direction == "none". Maps from: htf_neutral. "wait"
X.none HTF bias is directional but no phase block claimed the bar — gap / fallthrough. Maps from: htf_resolve_unclassified. Emits blocked_transition_reason, blocked_transition_context, and a compact blocked_phase_candidates map. "wait"
X.thesis_over A-phase finale confirmed: objective progress reached the configured maturity threshold (phase_a_thesis_matured). Post-cycle terminal; budget spent. "skip"
_phase_x() factory (current) hypothesis.py — _phase_x 
def _phase_x(self, phase_sub_status, reason, required_evidence, invalidation,
             ts, debug, range_reason=None) -> Hypothesis:
    extra = {
        "range_reason":   range_reason,
        "budget_policy":  "preserve_spent_budget",
    } if range_reason is not None else {}
    return Hypothesis(
        phase="X",
        status="watching",
        direction="none",
        entry_policy="skip" if phase_sub_status == "X.thesis_over" else "wait",
        target_policy="none",
        phase_sub_status=phase_sub_status,
        required_evidence=required_evidence,
        invalidation=invalidation,
        debug_facts={**debug, **extra},
    )

Only X.thesis_over gets entry_policy="skip" and the budget_policy extra key. All other sub-statuses get "wait".

X.none (htf_resolve_unclassified) is a gap, not a design intent. It fires when the DAG has a directional HTF bias but none of the E/D/C/B/A blocks claimed the bar. It should not be treated as equivalent to warmup or thesis.over. It now emits a single blocked_transition_reason plus a small context/candidate map; this is not an exhaustive error-code system.
X.none fallthrough debug hypothesis.py — _x_none_debug 
debug_facts = {
  "blocked_transition_reason": "directional_bias_but_no_phase_claimed",
  "blocked_transition_context": {
    "htf_phase": "pullback_confirmed",
    "previous_phase": None,
    "active_phase_e_direction": "none",
  },
  "blocked_phase_candidates": {
    "DAG": "pullback_confirmed_without_active_phase_context",
    "E":   "pullback_confirmed_hold_requires_previous_phase_E",

}

The candidate map is intentionally small. It records nearby DAG blockers that explain the fallthrough, such as missing active phase context, direct E→B blocking, or A requiring B origin. It should stay readable in replay, not become a complete matrix of every possible failed gate.

02 Phase X — thesis.over

Phase X is the terminal DAG node. It fires after _phase_a_finale() confirms the Phase A objective progress reached the configured maturity threshold. The full E→D→C→B→A cycle is mature; no fresh trades should be assumed near the HTF P/D objective. Phase X holds until a fresh HTF structural epoch or new Phase E seeking state resets the P/D range episode.

Entry gate — _phase_a_finale()

_phase_a_finale() computes direction-neutral objective progress from the current HTF bar and the active HTF P/D range. The default threshold is configured in replay.hypothesis.phase_a.objective_progress_threshold as 0.90. On the maturity bar the DAG transitions A.watch → X.thesis_over.

# Phase A objective maturity facts
phase_a_objective_progress_pct        float   # measured journey progress, e.g. 90.0
phase_a_objective_progress_threshold  float   # config ratio, default 0.90
phase_a_objective_progress_threshold_pct float # threshold as percent, default 90.0
phase_a_thesis_matured                bool    # True when progress reaches threshold

# Exact-touch EC facts are still emitted when price actually reaches the objective:
phase_a_finale_touched        bool    # True when HTF bar reached objective price
phase_a_finale_closed_beyond  bool    # True when HTF bar closed past objective
phase_a_finale_rule           str     # "bullish_phase_a_close_above_htf_pd_objective" etc.
phase_a_finale_objective      float   # the price level (EC emits a price, not a zone_id)
Dispatch sketch hypothesis.py — A block 
finale = self._phase_a_finale(snapshot, direction, htf, current_bar)
debug.update(finale)
if finale.get("phase_a_thesis_matured"):
    return self._phase_x(
        phase_sub_status="X.thesis_over",
        required_evidence=["phase_a_thesis_matured"],
        range_reason="phase_a_thesis_matured",
    )

The A block checks maturity before weaken/recovery exits. Exact objective touch remains visible through phase_a_finale_touched, but it is not required for thesis-over.

phase_a_thesis_matured is the DAG trigger. phase_a_finale_touched is exact-touch evidence and phase_a_shadow_objective_touched is the sticky shadow flag for Layer 5 quality logic.
