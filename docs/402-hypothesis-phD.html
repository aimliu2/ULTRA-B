Phase D — Simplify

TL;DR — Phase D is simplified to two nodes: D.watch and D.speculation. All prior sub-statuses superseded. D.watch opens from E.pullback_developing on the first LTF pro-HTF iChoCh (SC06 — internal ITR pivot break) — the structural initiation of the bounce; falls back to macro ChoCh (SC01/SC02) if no iChoCh preceded it (squeeze edge case, see §01). D.speculation fires when that pro attempt fails: a LTF counter-HTF iChoCh (SC06, internal ITR pivot break) or counter SB confirms the pullback is resuming. D.speculation → C.pullback on a fresh LTF orderflow MSS leg. Shadow thesis carries three epoch-scoped snapshots: phase_d_choch_1 (D.watch trigger), phase_d_pro_attempt (bounce quality while in D.watch), phase_d_choch_2 (D.speculation trigger). Layer 5 reads all three: choch_2 for SL placement (falls back to choch_1), pro-attempt for trade quality grading.

00 Core Concept

Phase D is the boundary where the system stops treating a counter move as soft Phase E memory and starts treating it as a global reaction process. It is still pre-entry — Layer 5 does not trade Phase D directly; D names the reaction and prepares the C pullback path.

The sub-status model is simplified to two nodes. Prior variants (HTF_reaction_point, LTF_reaction_point, htf_pd_grab_reclaim_test, htf_eq_grab_reclaim_test) are superseded. Evidence type — what zone or liquidity level was involved — lives in debug snapshots, not in the node name.

01 Phase D — New Model

Two nodes: D.watch and D.speculation. Both originate from E.pullback_developing. D.watch is the structural initiation of the pro-HTF bounce: it opens the moment the LTF prints its first pro-HTF iChoCh (SC06, internal ITR pivot break — earlier/finer signal) or falls back to macro ChoCh (SC01/SC02) in a squeeze edge case. D.speculation fires when that bounce fails — a LTF counter iChoCh or counter SB confirms the pullback is resuming. The distinction is directional commitment: D.watch = LTF bouncing with HTF; D.speculation = LTF resuming against HTF.

D.watch E.pullback_developing → DAG

Opens on the first LTF pro-HTF iChoCh after E.pullback_developing has started. The primary signal is last_isc (SC06 itrIChoCh, breakDirection == pro-HTF direction) — an internal ITR pivot break that fires before the macro range boundary is reached. Falls back to last_sc (macro SC01/SC02 ChoCh) when no iChoCh precedes the macro flip (squeeze edge case — see callout below).

Phase E shadow records whatever the bounce encountered — HTF SD zone, LTF SD zone, liquidity pool — as the pro_attempt context. D copies that into phase_d_pro_attempt at watch-open. A shallow bounce with nothing encountered is still valid; Layer 5 grades quality, not the gate. The structure engine may enter pullback_confirmed state during D.watch — this enables the counter SB path for D.speculation. 

Terminology — all structural events in Phase D are LTF events from the LTF structureEngine.
HTF provides only the directional reference: which LTF direction is counter, which is pro.

  LTF iChoCh  = LTF internal pivot break  — SC06 itrIChoCh, last_isc, is_internal=True
                                             fires against confirmed ITR pivot level (PE03/PE04)
                                             finer / earlier signal than macro ChoCh
  LTF ChoCh   = LTF macro structural flip — SC01/SC02, last_sc, choch=True
                                             fires against wick-defined range boundary
  LTF SB      = LTF structural break      — SC01/SC02, last_sc, choch=False  (BOS continuation)
  BoS / MSS   = orderflow engine events   — separate lineage, not from structureEngine

Gate (D.watch opens):
  phase_e_shadow_node == "E.pullback_developing"
  phase_e.pro_attempt_seen == True
    PRIMARY:  last_isc.breakDirection == pro-HTF direction   ← SC06 iChoCh, internal ITR pivot break
    FALLBACK: last_sc.breakDirection  == pro-HTF direction   ← SC01/SC02 macro ChoCh (squeeze edge case)
    ← pro_attempt_seen timestamp > pullback_developing_entered_at

Shadow stamps choch_1 snapshot (the pro-HTF ChoCh that opened D.watch):
  phase_d_choch_1.trigger_type       "choch"
  phase_d_choch_1.timestamp
  phase_d_choch_1.level              ← pro-HTF ChoCh level (e.g. prior range_high for long)
  phase_d_choch_1.structure_event_id | None
  phase_d_choch_1.source_level_id    | None
  phase_d_choch_1.ltf_sd_zone_id     | None
  phase_d_choch_1.ltf_sd_zone_high   | None
  phase_d_choch_1.htf_sd_zone_id     | None
  phase_d_choch_1.htf_sd_zone_high   | None
  phase_d_choch_1.liquidity_pool_id  | None   ← ITR level pool if present at bar
  phase_d_choch_1.liquidity_level    | None

Shadow carries pro_attempt snapshot (copied from Phase E at D.watch entry, then refined if later evidence appears):
  phase_d_pro_attempt.htf_sd_zone_id    | None   ← HTF SD zone reached during bounce
  phase_d_pro_attempt.htf_sd_zone_high  | None
  phase_d_pro_attempt.ltf_sd_zone_id    | None   ← LTF SD zone formed or tapped
  phase_d_pro_attempt.ltf_sd_zone_high  | None
  phase_d_pro_attempt.liquidity_pool_id | None   ← liquidity level tagged during bounce
  phase_d_pro_attempt.liquidity_level   | None

Squeeze edge case (fallback trigger):
  General case: ITR pivot sits inside PD range → SC06 iChoCh fires first (finer, earlier)
                → macro SC01/SC02 follows later if price continues → D.watch already open
  Edge case:    PD range contracted inside ITR high/low (squeezed)
                → PD range boundary is tighter than the ITR pivot level
                → SC01/SC02 fires before SC06 ever gets a chance
                → fallback to last_sc correctly catches it
                → Discovered in replay: EURUSD 15m/4h 2026-01-28T22:30 UTC

Holds while:
  no counter-HTF structural iChoCh or counter SB
  HTF has not closed above PD range

Exits:
  → D.speculation   first counter-HTF structural event fires (see D.speculation gate below)
  → E.seeking       HTF close above PD range (expansion resumed)
D.speculation D.watch → DAG

D.speculation has two entry paths. Path A: a LTF counter-HTF iChoCh fires (SC06, last_isc, internal ITR pivot break) — the bounce has failed and the LTF breaks a confirmed ITR low/high back counter-HTF. More sensitive than waiting for a full macro ChoCh. Path B: a LTF counter SB fires (macro, last_sc, choch=False) after the LTF structure engine enters pullback_confirmed during D.watch — bounce was structurally quiet. Both paths stamp the choch_2 snapshot. The same-bar shortcut is unreachable: LTF structureEngine caps at one SC event per bar, so D.watch always holds for at least one bar.

D.speculation → C.pullback on LTF orderflow MSS: the anchor swing low (orderflow engine, formed after the D.watch trigger) is broken on a fresh leg after the pro-attempt. Bounce height is not a condition. The consumed_leg_id prevents the same MSS event that opened E.pullback_developing from also triggering C. 

Gate — Path A (LTF counter-HTF iChoCh — bounce failed):
  current phase_sub_status == "watch"
  ltf_counter_choch_seen == True
    ← SC06 itrIChoCh, last_isc, break_direction == counter-HTF direction
    ← internal ITR pivot break — LTF closed through a confirmed PE04 low (bullish) or PE03 high (bearish)
    ← fires before macro ChoCh in normal conditions (ITR pivot sits inside PD range)
    ← source_store == "structure_isc" when triggered by SC06
    ← stronger signal: bounce failure visible at pivot level without waiting for range boundary break

Gate — Path B (LTF counter SB after pullback_confirmed):
  current phase_sub_status == "watch"
  ltf_counter_sb_seen == True             ← EC fact from ltf_counter_choch candidate (last_sc, macro)
    ← SC01/SC02, last_sc, choch=False, break_direction == counter-HTF direction
    ← LTF structure engine phase == "pullback_confirmed"   ← bounce recorded via ITR/SD pivot, no SC event yet
    ← LTF BOS: LTF counter-HTF range floor extended; no bias flip
    ← weaker signal: bounce was structurally quiet; only ITR/SD pivot registered
    ← source_store == "structure_sequence"

Shadow stamps choch_2 snapshot (same shape as choch_1):
  phase_d_choch_2.trigger_type       "choch" | "sb"   ← records which path fired
  phase_d_choch_2.timestamp
  phase_d_choch_2.level              ← ChoCh floor (Path A) or BOS floor (Path B)
  phase_d_choch_2.ltf_sd_zone_id     | None
  phase_d_choch_2.ltf_sd_zone_high   | None
  phase_d_choch_2.htf_sd_zone_id     | None
  phase_d_choch_2.htf_sd_zone_high   | None
  phase_d_choch_2.liquidity_pool_id  | None
  phase_d_choch_2.liquidity_level    | None

Layer 5 SL resolution (reads choch_2 first, falls back to choch_1):
  HTF SD zone present  →  htf_sd_zone_high  + 10 pips
  LTF SD zone present  →  ltf_sd_zone_high  + 20 pips
  liquidity grab       →  liquidity_level   + buffer
  nothing              →  level             + 20 pips (static)

Layer 5 quality grading (reads phase_d_pro_attempt):
  any field non-None  →  meaningful bounce — tighter SL, fuller size
  all None            →  shallow grind — acceptable, lower quality

→ C.pullback:
  ltf_counter_orderflow_mss_watch == True       ← orderflow MSS (anchor swing broken)
  AND orderflow_leg_id != phase_d_shadow.consumed_leg_id

→ E.stalling:   SL hit (price absorbed counter attempt, HTF not above PD range)
→ E.seeking:    HTF close above PD range
D.watch holds through the pro-HTF re-test intentionally. A LTF flicker back above the ChoCh level does not invalidate it — the HTF close above PD range is the only hard invalidation within Phase D.

02 Exit Paths

All D exits are driven by EC signals from two separate engines. Structural signals (ChoCh, counter SB) drive forward transitions within D. Orderflow MSS drives the D→C exit. HTF close above PD range is the invalidation path for both nodes.

Full exit map DAG 
D.watch exits:
  → D.speculation   Path A: ltf_counter_choch_seen     (SC06 iChoCh counter-HTF: bounce failed at ITR pivot)
  → D.speculation   Path B: ltf_counter_sb_seen        (macro SB counter-HTF, choch=False, pullback_confirmed)
  → E.seeking       HTF close above PD range (expansion resumed)

  Note — D.watch hangs on a BOS-only pullback if pullback_confirmed has not fired yet.
  Path B resolves this: once the structure engine records the bounce (pullback_confirmed),
  the next counter SB promotes to D.speculation without requiring a full iChoCh.

D.speculation exits:
  → C.pullback      ltf_counter_orderflow_mss_watch fires on a fresh leg
                    (orderflow engine: anchor swing low broken after pro-attempt)
                    AND orderflow_leg_id != phase_d_shadow.consumed_leg_id
  → E.stalling      SL hit — price absorbed the counter attempt
                    (HTF not above PD range; expansion not resumed)
  → E.seeking       HTF close above PD range

Both nodes — epoch boundary:
  → shadow cleared, DAG resets to E.seeking
D.speculation → E.stalling on SL hit. Shadow retains all three snapshots (phase_d_choch_1, phase_d_pro_attempt, phase_d_choch_2) for the full epoch — overwritten if a new D.watch opens later in the same epoch. All three survive into C and B for Layer 5 to read at entry time.

03 Algorithm

Implemented in Layer 4 as a shadow-node state machine. The archived four-sub-status selector is disabled and preserved only in §06 for migration reference. EC emits structural facts through ltf_counter_choch and reaction/MSS facts through phase_e_context; Layer 4 owns D-node selection. Structural terms (ChoCh, SB) come from structureEngine; orderflow terms (MSS) come from orderflow. 

# ── EC facts in scope ──────────────────────────────────────────────────────
# phase_e.pro_attempt_seen      — PRIMARY:  last_isc SC06 iChoCh, break_direction==pro-HTF direction
#                                  FALLBACK: last_sc  SC01/SC02 macro ChoCh (squeeze edge case)
#                                  D.watch trigger. iChoCh fires first in normal market structure.
# ltf_counter_choch_seen        — SC06 itrIChoCh, last_isc, break_direction==counter-HTF direction
#                                  Internal ITR pivot break — bounce failed at pivot level
#                                  D.speculation Path A trigger (source_store="structure_isc").
# ltf_counter_sb_seen           — SC01/SC02 macro SB, last_sc, break_direction==counter-HTF direction
#                                  phase=="pullback_confirmed", BOS continuation (no bias flip)
#                                  D.speculation Path B trigger (source_store="structure_sequence").
# ltf_counter_orderflow_mss_watch — orderflow MSS: LTF orderflow anchor swing broken on fresh leg
#                                  LTF orderflow engine — distinct from LTF structureEngine SB/ChoCh

# ── D.watch gate ───────────────────────────────────────────────────────────
if (
    phase_e_shadow_node == "E.pullback_developing"
    and phase_e_pro_attempt_seen   # PRIMARY: SC06 iChoCh pro-direction; FALLBACK: SC01/SC02 (squeeze)
):
    copy phase_e_pro_attempt → phase_d_pro_attempt
    stamp phase_d_choch_1(trigger_type="choch", level=pro_choch_level, at=pro_attempt_started_at)
    → D.watch

# ── D.watch — each bar ─────────────────────────────────────────────────────
if phase_sub_status == "watch":
    refine phase_d_pro_attempt if new bounce-quality facts appear
    if htf_close_above_pd_range:
        → E.seeking
    elif ltf_counter_choch_seen:     # Path A: SC06 iChoCh counter-HTF (last_isc, internal pivot break — bounce failed)
        stamp phase_d_choch_2(trigger_type="choch", level=choch_level, ...)
        → D.speculation
    elif ltf_counter_sb_seen:        # Path B: counter SB (choch=False, pullback_confirmed)
        stamp phase_d_choch_2(trigger_type="sb", level=sb_level, ...)
        → D.speculation

# ── D.speculation — each bar ───────────────────────────────────────────────
if phase_sub_status == "speculation":
    if (
        ltf_counter_orderflow_mss_watch          # orderflow engine: anchor swing low broken
        and ltf_counter_orderflow_leg_id != consumed_leg_id  # fresh leg (not same MSS that opened E.pullback_dev)
    ):
        set consumed_leg_id = ltf_counter_orderflow_leg_id
        → C.pullback
    elif sl_hit:
        → E.stalling
    elif htf_close_above_pd_range:
        → E.seeking

05 Open Issues

Archived · Completed

Phase D upgraded to iChoCh (SC06) — squeeze edge case documented. D.watch and D.speculation both upgraded to use SC06 itrIChoCh (last_isc) as primary trigger instead of macro SC01/SC02 ChoCh (last_sc). In normal market structure, ITR pivot sits inside the PD range — SC06 fires first (finer/earlier signal). Discovered edge case: when the PD range contracts inside the ITR high/low (squeezed relationship), the range boundary is tighter than the ITR pivot level and macro SC01/SC02 fires before SC06 has a chance. Fallback to last_sc handles this correctly. Confirmed in replay: EURUSD 15m/4h 2026-01-28T22:30 UTC.

06 Old Model (Archived)

The prior Phase D had four sub-statuses. Superseded by the D.watch / D.speculation model. Preserved here for reference during the migration.

D.HTF_reaction_point Archived

The HTF path required the expansion to have stopped and an opposing HTF S/D zone to be tapped, resolved, or confirmed as a pullback. 

EC:
  pattern = htf_counter_reaction
  status  = ready
  normal_ready path:
    htf_pd_stopped_expanding       ← expansion stopped
    htf_opposing_sd_reaction       ← HTF zone tapped | resolved | confirmed pullback
    ltf_counter_sd_created         ← counter LTF zone exists
  special_ready path (edge case):
    new_htf_extreme                ← strict new extreme
    no htf_opposing_sd_reaction
    ltf_bias_counter_htf

DAG:
  if previous/current E and phase_d_ready and no liquidity candidate:
    phase = D
    phase_sub_status = HTF_reaction_point
D.LTF_reaction_point Archived

A lighter path born directly from E.pullback_developing using only LTF structural evidence. Did not require HTF expansion stop, opposing HTF zone, or any liquidity event. 

Gate:
  previous_phase not in {"B", "A"}
  phase_e_shadow_node == "E.pullback_developing"
  ltf_counter_choch_seen == True

DAG:
  phase = D
  phase_sub_status = LTF_reaction_point
  origin_node = E.pullback_developing
