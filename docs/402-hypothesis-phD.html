Phase D — Simplify

TL;DR — Phase D has one DAG node: D.watch. All prior sub-statuses superseded. D.watch opens from E.pullback_developing on the first pro-HTF macro ChoCh (last_sc, SC01/SC02). D.watch + counter MSS (fresh leg) → C.pullback (c_shadow.origin_node = "D.watch_mss") — the pro-attempt definitively failed. iChoCh (last_isc, SC06), counter SB, and entry timing are Layer 5 territory — see docs/501-entry.html. 00 Core Concept

Phase D is the boundary where the system stops treating a counter move as soft Phase E memory and starts treating it as a global reaction process. It is still pre-entry — Layer 5 does not trade Phase D directly; D names the reaction and prepares the C pullback path.

The sub-status model is simplified to two nodes. Prior variants (HTF_reaction_point, LTF_reaction_point, htf_pd_grab_reclaim_test, htf_eq_grab_reclaim_test) are superseded. Evidence type — what zone or liquidity level was involved — lives in debug snapshots, not in the node name.

01 Phase D — Current Model

One DAG node: D.watch. D.speculation has been removed from the DAG — its mechanics (iChoCh, counter SB, SL reference) are now Layer 5 entry confirmation. D.watch opens from E.pullback_developing on the first pro-HTF macro ChoCh (last_sc, SC01/SC02). D.watch + counter MSS → C.pullback. Everything between those two transitions is Layer 5 territory.

D.watch E.pullback_developing → DAG

Opens from E.pullback_developing on the first pro-HTF macro ChoCh (last_sc, SC01/SC02). iChoCh (last_isc, SC06) is Layer 5 territory — see docs/501-entry.html.

Shadow carries two epoch-scoped snapshots: choch_1 (the SC01/SC02 that opened D.watch — Layer 5 reads level for SL reference) and pro_attempt (quality metadata accumulated while holding in D.watch). A shallow bounce with nothing encountered is valid; Layer 5 grades quality, not the gate. 

Gate (D.watch opens — standard):
  current hypothesis is not X.thesis_over
  phase_e_shadow_node == "E.pullback_developing"
  phase_e.pro_attempt_seen == True
    last_sc.breakDirection == pro-HTF direction   ← SC01/SC02 macro ChoCh
    ← pro_attempt_seen timestamp > pullback_developing_entered_at
  phase_d_entry = "E.pullback_developing"

Shadow carries (refined while holding):
  watch_entered_at        cursor_time at open
  consumed_leg_id         copied from phase_e.source_orderflow_leg_id at open
  pro_attempt.htf_reaction_status   ← htf_counter_reaction EC status
  pro_attempt.ltf_story_status      ← ltf_counter_story EC status

Holds while:
  counter MSS has not fired (or same MSS leg as consumed_leg_id)
  HTF has not closed above PD range

Exits:
  → C.pullback      counter MSS fires on fresh leg (ltf_counter_orderflow_mss_watch,
                    AND orderflow_leg_id != consumed_leg_id)
                    c_shadow.origin_node = "D.watch_mss"   ← Layer 5 Path C gate
  → E.seeking       HTF close above PD range (expansion resumed)

D.watch hold — debug output (every bar while holding):
  phase_d_node                       "D.watch"
  phase_e_shadow_htf_reaction_seen   bool  ← PhaseEShadow.htf_reaction_seen
    sticky: set once at E.stalling zone tap, never cleared within epoch
    Layer 5 reads this for htf_zone_context analysis tagging; immune to EC clearing on 4H close

02 Exit Paths

D.watch has two exits: counter MSS (→ C.pullback) and HTF expansion (→ E.seeking). Structural iChoCh and SB signals are no longer DAG transitions — they are Layer 5 signals consumed after the snapshot is read.

Full exit map DAG 
D.watch exits:
  → C.pullback      ltf_counter_orderflow_mss_watch fires on a fresh leg
                    (orderflow engine: anchor swing low broken after pro-attempt)
                    AND orderflow_leg_id != phase_d_shadow.consumed_leg_id
                    c_shadow.origin_node = "D.watch_mss"   ← Phase C provenance
  → E.seeking       HTF close above PD range (expansion resumed)

D.watch — epoch boundary:
  → shadow cleared, DAG resets to E.seeking

Layer 5 reads from D.watch snapshot (not DAG transitions):
  ltf_counter_ichoch_isb_sequence_seen← internal counter iChoCh → internal counter iSB → D.watch_pathA
  ltf_counter_sequence_trade_direction← must match the Layer 5 trade direction
  ltf_counter_sequence_isb_event_at   ← must be fresh after D.watch opened
  ltf_counter_sb_seen                 ← macro SB counter-HTF → quality context
  phase_e_shadow_htf_reaction_seen    ← sticky bool: htf_zone_context analysis tag
  phase_d_shadow.pro_attempt          ← quality grading context
Shadow retains choch_1 and pro_attempt for the full epoch — overwritten only if a new D.watch opens later in the same epoch. Both survive into C and B for Layer 5 to read at entry time.

03 Algorithm

Implemented in Layer 4 as a shadow-node state machine. EC emits structural facts through ltf_counter_choch and reaction/MSS facts through phase_e_context; Layer 4 owns D-node selection. Structural terms (ChoCh, SB) come from structureEngine; orderflow terms (MSS) come from orderflow. iChoCh and counter SB facts are passed through to Layer 5 via snapshot but do not trigger DAG transitions. 

# ── EC facts in scope ──────────────────────────────────────────────────────
# phase_e.pro_attempt_seen        — last_sc SC01/SC02 macro ChoCh, break_direction==pro-HTF direction
#                                   D.watch trigger.
# ltf_counter_ichoch_isb_sequence_seen — internal counter iChoCh → internal counter iSB (EX.entry)
#                                        Layer 5 D.watch_pathA trigger. Not a DAG gate.
# ltf_counter_sb_seen                  — major counter SB quality context. Not a D entry path.
# ltf_counter_orderflow_mss_watch — orderflow MSS: LTF orderflow anchor swing broken on fresh leg
#                                   D.watch → C.pullback gate.
# new_htf_extreme                 — phase_e_context: price made a new HTF extreme (expansion resumed)
#                                   D.watch → E.seeking gate (checked before MSS).

# ── D.watch gate — standard (pro-attempt macro ChoCh) ──────────────────────
if (
    phase_e_shadow_node == "E.pullback_developing"
    and phase_e_pro_attempt_seen   # last_sc SC01/SC02 macro ChoCh pro-direction
):
    d_shadow.node = "D.watch"
    d_shadow.watch_entered_at = pro_attempt_started_at
    d_shadow.consumed_leg_id = phase_e_shadow.source_orderflow_leg_id
    emit phase_d_entry = "E.pullback_developing"
    → D.watch

# ── D.watch — each bar ─────────────────────────────────────────────────────
if phase_sub_status == "watch":
    if new_htf_extreme:              # phase_e_context: expansion resumed — checked first
        → E.seeking
    elif (
        ltf_counter_orderflow_mss_watch          # orderflow engine: anchor swing broken
        and ltf_counter_orderflow_leg_id != consumed_leg_id  # fresh leg
    ):
        set consumed_leg_id = ltf_counter_orderflow_leg_id
        c_shadow.origin_node = "D.watch_mss"   # Layer 5 Path C reads this
        → C.pullback
    else:
        accumulate pro_attempt quality metadata (htf_reaction_status, ltf_story_status)
        emit phase_e_shadow_htf_reaction_seen = shadow_thesis.phase_e.htf_reaction_seen
        hold D.watch  # Layer 5 reads iChoCh/iSB from snapshot for entry confirmation

05 Open Issues

Open · Migration Deferred

HTF S/D zone context demoted from entry gate to analysis tag — completed. phase_e_shadow_htf_reaction_seen and phase_d_shadow_htf_zone_seen now feed Layer 5's htf_zone_context regime tag only. They do not select a separate D path and do not provide the SL anchor. Current Phase D entry is D.watch_pathA only.

Archived · Completed

Phase D upgraded to iChoCh (SC06) — squeeze edge case documented. D.watch and D.speculation both upgraded to use SC06 itrIChoCh (last_isc) as primary trigger instead of macro SC01/SC02 ChoCh (last_sc). In normal market structure, ITR pivot sits inside the PD range — SC06 fires first (finer/earlier signal). Discovered edge case: when the PD range contracts inside the ITR high/low (squeezed relationship), the range boundary is tighter than the ITR pivot level and macro SC01/SC02 fires before SC06 has a chance. Fallback to last_sc handles this correctly. Confirmed in replay: EURUSD 15m/4h 2026-01-28T22:30 UTC.

Archived · Completed

ltf_itr_level_run gate dropped — not a Layer 4 gate condition. Layer 4 reads compiled EC facts only. The existing probe compiled terms answer questions about HTF PD range geography (ltf_probe_outside_htf_pd_range, ltf_probe_at_htf_opposing_zone), not LTF ITR structure extremes. A new EC fact would be required, but the ITR-level gate itself is redundant: the D boundary is now controlled by lineage and post-E pro-attempt separation. ITR level context (which ITR high/low was active at the D.watch bar) is captured as an optional field in phase_d_choch_1 snapshot for Layer 5 quality grading — not as a gate blocker. D.watch opens on the first LTF pro-HTF ChoCh (the bounce initiation). The old counter-ChoCh-after-pro-attempt model was incorrect — the pro ChoCh IS the D.watch trigger, not a precondition for a later counter event.

Archived · Completed

D.watch → C.pullback trigger: LTF orderflow MSS on fresh leg. Gate: ltf_counter_orderflow_mss_watch == True and ltf_counter_orderflow_leg_id != phase_d_shadow.consumed_leg_id. The protected orderflow anchor is broken after the pro-attempt. Bounce height is not a condition — a shallow grind-down with a minimal bounce still fires. consumed_leg_id prevents the same MSS that opened E.pullback_developing from also triggering C.

Archived · Completed

Same-bar D.speculation shortcut: unreachable — structureEngine caps at one SC event per bar. The early return at _bos_range() after the first emit means two ChoChs on the same bar cannot occur. D.watch always holds for at least one bar minimum. The "same-bar" concern is moot. The 1-bar lag is accepted: close is the system's commitment signal. The accepted 1-bar lag for volatile bars is documented in §02 exit map.

06 Old Model (Archived)

The prior Phase D had four sub-statuses. Superseded by the D.watch-only model. Preserved here for migration reference.

D.HTF_reaction_point Archived

The HTF path required the expansion to have stopped and an opposing HTF S/D zone to be tapped, resolved, or confirmed as a pullback. 

EC:
  pattern = htf_counter_reaction
  status  = ready
  normal_ready path:
    htf_pd_stopped_expanding       ← expansion stopped
    htf_opposing_sd_reaction       ← HTF zone tapped | resolved | confirmed pullback
    ltf_counter_sd_created         ← counter LTF zone exists
  special_ready path (edge case):
    new_htf_extreme                ← strict new extreme
    no htf_opposing_sd_reaction
    ltf_bias_counter_htf

DAG:
  if previous/current E and phase_d_ready and no liquidity candidate:
    phase = D
    phase_sub_status = HTF_reaction_point
D.LTF_reaction_point Archived

A lighter path born directly from E.pullback_developing using only LTF structural evidence. Did not require HTF expansion stop, opposing HTF zone, or any liquidity event. 

Gate:
  previous_phase not in {"B", "A"}
  phase_e_shadow_node == "E.pullback_developing"
  ltf_counter_choch_seen == True

DAG:
  phase = D
  phase_sub_status = LTF_reaction_point
  origin_node = E.pullback_developing
