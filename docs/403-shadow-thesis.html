Layer 4 · memory and restart semantics

403 - Shadow Thesis Persistence

TL;DR - Shadow Thesis is the hiker's journal. Runtime now restores a saved journal when epoch/direction/anchors are valid, rebuilds it from hidden Layer 4 bootstrap bars when persistence is unavailable, and only then uses conservative terrain relocation. X.none is no longer the normal answer to a missing journal; it is reserved for contradictory or unsupported terrain.
→ Back to 401 Hypothesis → Open DAG Map → Related: Phase X Startup States

00 Core Concept

The DAG is the map, EC and Layer 3 are the terrain report, and Shadow Thesis is the local journal. The journal records what Layer 4 has already committed to: which pullback leg was consumed, which pro attempt opened D, which C origin is active, which B commitment extreme is protected, and which A weaken/recovery memory is still valid.

A restart must first decide whether the journal is still trustworthy. If it is trustworthy, the system resumes it. If the journal is missing, the system should rebuild it from hidden bars when enough local history exists. If it cannot be rebuilt, the system relocates from current terrain. Resetting to E.seeking or falling to X.none is not the default.

!
The 2026-02-02 startup failure happened because Layer 3 terrain was warmed but Layer 4 woke up blank. Runtime now covers this path: 2026-02-02T00:00:00+00:00 runs 300 hidden bootstrap bars and reaches C.pullback before the first visible bar.
!
The validity discriminator is the epoch, not calendar time. A saved journal from three weeks ago is valid if htf_pd_epoch_id is unchanged — the hiker slept a long time but woke up in the same valley. A saved journal from Friday is invalid if a new HTF BOS/ChoCh fired over the weekend — the terrain reset, so the old commitment anchors are void regardless of how little time passed.
Discretionary → Algorithm mapping field journal

Discretionary: if the hiker slept on the hill, his journal still describes the trip. He resumes from the same waypoint unless the terrain proves the trip is invalid.

Algorithm: if persisted HypothesisClassifierState has the same htf_pd_epoch_id as current terrain and no hard invalidation fires, restore current_hypothesis and shadow_thesis.

Discretionary → Algorithm mapping lost journal

Discretionary: if the hiker lost the journal, he walks the recent trail if footprints are visible; if not, he reads the terrain and finds the nearest defensible map node.

Algorithm: run hidden Layer 4 bootstrap over recent bars. If that does not produce a valid journal, use direction-checked EC candidates and Layer 3 snapshots to relocate to the deepest reconstructable node.

Shadow Thesis is Layer 4 memory. It may be persisted, restored, rebuilt, or cleared, but EC must never write it. EC says what the market did; Shadow Thesis says what Layer 4 believed and committed to.

01 Restart Modes

Startup has three implemented recovery modes before X.none. The modes are ordered by trust. A saved journal is more faithful than hidden bootstrap; hidden bootstrap is more faithful than current-terrain relocation.

ModeUse whenExpected outcome
resume_saved_journal Planned restart, same htf_pd_epoch_id, persisted state exists. Resume A/B/C/D/E from stored current_hypothesis and shadow_thesis.
hidden_layer4_bootstrap No usable saved journal, but enough bars exist before visible start. Replay hidden bars through EC + hypothesis to reconstruct journal.
terrain_relocation Journal missing and bootstrap unavailable or inconclusive. Choose deepest reconstructable node from current EC/L3 terrain.
X.none Directional terrain is contradictory or insufficient to select even E/C. Emit diagnostic gap, not a normal startup path.
Case 1 - slept on the hill planned maintenance

Friday ends in A.watch under a bullish HTF epoch. The market closes, the process stops, and Monday reopens inside the same HTF epoch. The correct startup path is to restore the persisted journal and continue A.watch, not to relocate to C.

Case 2 - journal missing but recent trail exists bootstrap

The process starts at 2026-02-02T00:00:00+00:00 with no saved journal. The implemented hidden Layer 4 bootstrap walks 300 prior 15m bars and arrives at C.pullback short before the first visible bar.

Case 3 - fell into the river terrain relocation

If saved state and bootstrap cannot reconstruct the journal, the system reads current terrain. It may relocate to A or B only if the required commitment anchors can be reconstructed. Otherwise it should stop at the deepest honest node, usually C inside HTF pullback terrain or E inside HTF open terrain.

Case 4 - long downtime, epoch unchanged persist still valid

The system was down for days or weeks. On restart, EC warms up and reads htf_pd_epoch_id from current terrain. If it matches the saved journal's epoch, the persisted journal is valid regardless of how long the system was offline. The hiker slept a long time but woke up in the same valley — same trail, same commitment line. Restore the journal, run one classify pass, continue.

Uptime tracking is not needed. The epoch check already acts as the proxy: if the HTF structure did not reset, the old commitment anchors are still valid.

Case 5 - long downtime, epoch changed persist voided, reconstruct new epoch

A new HTF BOS/ChoCh fired while the system was down. The saved epoch no longer matches current terrain. Reject the persisted journal — the old commitment anchors are void — and fall to the hidden Layer 4 bootstrap. The bootstrap walks the 300 bars before start_time and reconstructs the journal for the new epoch.

On 15m LTF, 300 bootstrap bars = ~75 hours. If the new epoch's C→B entry fired within that window, the bootstrap reconstructs b_shadow.commitment_extreme_level correctly. If C→B fired earlier, the bootstrap starts mid-epoch without the commitment anchor and terrain relocation becomes the honest fallback.

Multiple epoch changes during downtime reduce to the same case: only the current epoch matters. All prior epochs are void by definition.

Do not use terrain relocation while a valid saved journal exists. A valid journal is the highest-fidelity source of Layer 4 memory.

02 Persistence Contract

Persistence must capture the whole Layer 4 state required to resume a hypothesis. Saving only the phase label is unsafe because B and A depend on commitment anchors stored in shadow fields.

Persist the classifier state HypothesisClassifierState

Persist hypothesis_id, phase_episode_id, previous_phase, htf_pd_epoch_id, active_phase_e_direction, current_hypothesis, and the full shadow_thesis tree.

Validate before restore epoch and direction

Restore only when current terrain still agrees with the saved epoch and direction. If current htf_pd_epoch_id differs, clear the journal and move to bootstrap or relocation.

Resume through normal DAG not raw carry

After restore, run one normal classification pass. This lets existing invalidation gates fire on the first live bar instead of blindly carrying stale state.

$$restore\_ok = saved.epoch = current.epoch \land saved.direction = current.direction \land \neg hard\_invalidated$$

Must-persist vs reconstructable fields

Not all shadow fields need to be persisted. Fields EC can provide fresh after Layer 3 warmup are reconstructable. Fields that are historical snapshots taken at a specific past bar are not.

FieldClassificationReason
b_shadow.commitment_extreme_level Must persist Snapshot of last_sc.breakLevel at C→B entry bar. last_sc may have advanced since then; Layer 3 only holds the current value.
b_shadow.commitment_extreme_event_id Must persist Historical event ID at C→B entry. Required to verify freshness gates in A.watch.
a_shadow.entered_at Must persist Timestamp of A entry bar. Layer 3 does not track when Layer 4 phase transitions fired.
a_shadow.pro_extreme_at_weaken Must persist Anti-oscillation gate — price at which A.watch weakened. Required to block premature recovery re-entry.
c_shadow.origin_node Must persist V1 vs V2 entry path. Determines which gates are valid inside C. Not derivable from current terrain.
c_shadow.entered_at Must persist Freshness floor for C-internal gates.
e_shadow.source_orderflow_leg_id Must persist MSS leg that opened E.pullback_developing. Required to gate D.watch entry on fresh leg only.
htf_pd_epoch_id Reconstructable EC reads this directly from HTF structure on every bar. Used for validation, not restoration.
active_phase_e_direction Reconstructable EC reads HTF bias on every bar. Confirmed from current terrain at restore time.
Current phase label (E/D/C/B/A) Reconstructable Derived from terrain relocation if persist fails. Not needed when persist succeeds because current_hypothesis is restored.
commitment_extreme_level cannot be sourced from Layer 3 on restart. It is a historical last_sc.breakLevel snapshot taken at C→B entry. Layer 3 only holds the current last_sc — if any new BOS/ChoCh fired between C→B entry and the restart, Layer 3's value is wrong. Pushing this to EC would also violate ownership: EC tracks what the market did, not what Layer 4 committed to.

03 Bootstrap Window

Runtime now splits startup context into Layer 3 terrain warmup and hidden Layer 4 bootstrap. The bootstrap slice runs hidden bars through HypothesisClassifier.classify() before the first visible replay bar.

Terrain warmup Layer 3 only

The first slice prepares structure, zones, orderflow, and liquidity. It does not write hypothesis memory.

Layer 4 bootstrap hidden history

The second slice runs full snapshots through EC and the hypothesis classifier. These bars are not visible replay bars, but they build the Shadow Thesis journal for the first visible bar.

window_bars = 500
warmup_bars = 200
hypothesis_bootstrap_bars = window_bars - warmup_bars

start_index = requested visible start

[start - 500, start - 300)  terrain warmup only
[start - 300, start)        hidden EC + Layer 4 bootstrap
[start]                     first visible bar with journal already built
window_bars should not only mean visible attention. For startup, it should define the complete context budget: terrain warmup plus hidden Layer 4 bootstrap plus the first visible bar.
!
On 15m LTF, 300 bootstrap bars = ~75 hours. This is the window available to reconstruct the current epoch's Shadow Thesis after a persist reject (epoch mismatch). If the new epoch's C→B entry fired within 75 hours of start_time, the bootstrap reconstructs b_shadow.commitment_extreme_level and all commitment anchors. If C→B fired earlier, the bootstrap starts mid-epoch and terrain relocation becomes the honest fallback. Extend hypothesis_bootstrap_bars if typical C→B onset on the HTF routinely exceeds this window.

04 Terrain Relocation

Terrain relocation is the fallback when no valid journal exists and hidden bootstrap cannot reconstruct one. The current implementation is conservative: it can relocate honestly to C or E, and rejects A/B/D with explicit reasons until their anchors/provenance can be reconstructed.

A relocation hardest

Target rule: accept only if A terrain is clear and B/A anchors are reconstructable: b_shadow.commitment_extreme_level, b_shadow.commitment_extreme_event_id, phase_a.entered_at, and any weaken/recovery memory needed by current substate. Current runtime rejects A relocation with an explicit diagnostic reason.

B relocation commitment required

Target rule: accept only if B watch terrain is clear and the B commitment extreme can be reconstructed from visible LTF structure. Current runtime rejects B relocation rather than guessing the commitment extreme.

C relocation terrain-readable

Accept when HTF is in pullback terrain and direction-checked ltf_counter_story / phase_e_context supports counter-pullback state. If selected POI exists, use armed; otherwise use watching.

E relocation expansion terrain

Accept when HTF is open or expansion-seeking and no deeper reconstructable pullback/continuation node is supported. Runtime commits E from the current EC/L3 facts.

Relocation priority:
  1. A only if B/A journal anchors can be reconstructed
  2. B only if commitment extreme can be reconstructed
  3. C when HTF pullback terrain is clear
  4. D only if D.watch provenance is reconstructable
  5. E when HTF open / expansion terrain is clear
  6. X.none only for contradiction, not missing journal
C is not the universal restart answer. It is the honest fallback inside HTF pullback terrain when deeper B/A commitment memory cannot be reconstructed.

05 Algorithm

The restart algorithm lives in DualSmcRuntime initialization and classifier state handling. Normal per-bar DAG logic remains focused on current-state transitions. 

# Runtime startup hierarchy implemented in DualSmcRuntime._init_engines()
def start_runtime(start_time):
    build_engines()

    saved = load_shadow_state(symbol, combo)
    if saved:
        warm_layer3_terrain(to_visible_start)
    if restore_saved_journal(saved, current_terrain):
        return start_visible_replay()

    rebuild_engines()
    warm_layer3_terrain(to_bootstrap_start)
    if run_hidden_layer4_bootstrap():
        return start_visible_replay()

    if relocate_from_current_terrain():
        return start_visible_replay()

    return start_visible_replay_with_x_none_diagnostics()
def restore_saved_journal(saved, terrain):
    if not saved:
        return False
    if saved.schema_version != CURRENT_SCHEMA:
        return False
    if saved.htf_pd_epoch_id != terrain.htf_pd_epoch_id:
        return False
    if saved.active_phase_e_direction != terrain.direction:
        return False
    if saved.current_hypothesis.phase in {"A", "B"} \
       and not saved.shadow_thesis.phase_b.commitment_extreme_level:
        return False

    classifier.state = deserialize(saved.classifier_state)
    # Let the next normal classify pass invalidate if needed.
    return True
def run_hidden_layer4_bootstrap():
    start = lower_start_index - hypothesis_bootstrap_bars
    for idx in range(start, lower_start_index):
        step_layer3_to(idx)
        payload = snapshot(classify=False)
        payload.evidence_candidates = ec.update(payload.context_snapshot)
        classifier.classify(payload)

    return classifier.state.current_hypothesis.phase in {"A", "B", "C", "D", "E"}
def relocate_from_current_terrain():
    candidates = [
        reject_a_until_anchors_reconstructable(),
        reject_b_until_commitment_reconstructable(),
        relocate_c_if_supported(),
        reject_d_until_watch_provenance_reconstructable(),
        relocate_e_if_supported(),
    ]
    selected = first_ready(candidates)
    if not selected:
        return False
    write_reconstructed_shadow(selected)
    commit_hypothesis(selected)
    return True

Edge Case

Weekend maintenance. Test coverage finds the last pre-gap bar and first post-gap bar from the local data. If epoch and anchors match, restart must use resume_saved_journal; fuller validation can wait for bot-layer operation.

Fallback

Missing or stale saved journal. Runtime rejects persisted state on schema mismatch, epoch mismatch, direction mismatch, or missing required anchors, then runs hidden bootstrap before terrain relocation.

Archived · Completed

Archived · Completed

Direction-sensitive EC reads. Layer 4 now uses direction-checked candidate reads for direction-sensitive E/C/D/B/A gates, which is required before terrain relocation can be trusted.
