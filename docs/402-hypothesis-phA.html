Layer 4 · Hypothesis · Phase A

Hypothesis Phase A — A.watch

TL;DR — Phase A is the pro-attempt finale: the node where the HTF thesis either confirms or fails. Two states: A.watch (active pro attempt) and A.watch_weaken (counter orderflow MSS weakens the pro attempt). A.watch exits: new HTF extreme → E.seeking; counter MSS → A.watch_weaken. A.watch_weaken exits: B commitment extreme breached → C.pullback (A.watch_weaken_invalidated); pro SC + new pro-side extreme (pro extreme advance gate) → recovery. The pro extreme advance gate blocks oscillation in ranging markets: recovery requires the pro-side LTF range extreme to have advanced past the level locked at weaken time. pro_attempt_weaken is sticky — Layer 5 raises the entry bar.
→ Phase B — B.watch → Phase C — Counter Pullback → DAG diagram

00 Overview

Phase A is the final node in the hypothesis DAG chain. It opens when B.watch receives a pro-HTF BoS (Break of Structure in the pro-HTF direction), indicating price has escaped the B zone and the pro side is attempting to drive toward the HTF P/D objective. Unlike C (which tracks the counter pullback) and B (which watches for the pro reclaim initiation), Phase A is purely about whether the pro-HTF attempt has enough structural conviction to confirm.

DAG position:

E.seeking → E.stalling → E.pullback_developing
  → D.watch
    → C.pullback              ← counter MSS confirmed; pullback leg active
        depth < 51%: → C.pullback_weaken
        depth ≥ 51%: → B.watch           ← pro-HTF SC; discount zone reached
          → C.pullback   ← counter MSS (reclaim failed)
          → A.watch      ← pro-HTF BoS after B; finale
              → A.watch_weaken  ← counter SC fires within A zone
              → B.watch         ← counter MSS (pro attempt failed)
              → E.seeking       ← new HTF extreme
Phase A is not a pullback depth node. The pullback depth gate lives at the C → B boundary (≥ 51%). No additional depth condition applies at B → A. The trigger is purely structural: a pro-HTF BoS after B.watch has been active.

01 Entry Gate — B → A

A.watch opens from the B block (previous_phase == "B", phase_sub_status == "watch") when a pro-HTF BoS fires on a fresh leg after B.watch opened. The gate reads from phase_e_context facts — no additional EC candidate is required for the DAG transition itself. The htf_pd_objective EC candidate is available to Layer 5 for entry permission decisions but is not a DAG gate.

Gate conditions

previous_phase == "B"  AND  phase_sub_status == "watch"
last_sc.breakDirection == pro-HTF direction     # pro BoS (SC01/SC02 or macro level)
last_sc.eventTimestamp > b_shadow.entered_at     # freshness: BoS after B.watch opened

There is no depth gate at this boundary. The 51% depth condition was already satisfied when C → B fired. B → A is triggered by the structural signal alone — price escaping the B zone in the pro-HTF direction.

Unlike C → B (which has an explicit depth gate), B → A has no depth precondition. Depth was already screened at C → B. The pro-HTF BoS freshness check (after b_shadow.entered_at) is the only gate — preventing a stale BoS that predates B.watch from firing A entry.

02 Shadow — PhaseAShadow

A.watch carries commitment pointers only. Layer 3 EC snapshot is the authoritative source for zone levels, orderflow direction, and prices. Layer 5 resolves IDs from the EC snapshot at decision time. Shadow fields are never zone levels or prices — only IDs, booleans, and timestamps.

PhaseAShadow fields

entered_at                   str | None   # cursor_time of B → A transition (freshness floor)
pro_attempt_weaken           bool         # True once A.watch_weaken fires (sticky)
pro_attempt_weaken_at        str | None   # timestamp of first weaken event
pro_extreme_at_weaken        float | None # range_high (long) / range_low (short) locked at weaken
                                          # recovery waits for pro-side extreme advance
recover_at                   str | None   # timestamp of weaken → watch recovery
phase_a_objective_touched    bool         # True once HTF P/D objective price touched (sticky)
phase_a_objective_touched_at str | None   # cursor_time of first exact objective touch

entered_at is set on the B → A transition bar and serves as the freshness floor for the A.watch_weaken trigger guard. Objective touch fields are updated each bar by _update_phase_a_shadow(), which reads from htf_pd_objective EC facts.

pro_extreme_at_weaken is captured at the moment A.watch → A.watch_weaken fires, from ltf.range_high (long) or ltf.range_low (short) at that exact bar. It is the pro extreme advance reference: recovery to A.watch is blocked until range_high > pro_extreme_at_weaken for long or range_low < pro_extreme_at_weaken for short, preventing oscillation in sideways markets where micro pro SCs fire without genuine structural progress.

pro_attempt_weaken is sticky once set — it is never reset within an A.watch episode. It signals to Layer 5 that the pro attempt lost conviction at least once. Layer 5 raises the entry bar accordingly: requires phase_a_shadow_objective_touched == True or b_shadow.liquidity_pool_run == True (carried from B) before permitting entry.

Shadow pointer fields accumulate monotonically — they are never cleared within an A.watch episode. pro_attempt_weaken is sticky: once a counter SC fires within A zone, the weaken flag stays set even after recovery. Layer 5 reads this as a permanent conviction reduction for this A episode.
Shadow update sketch hypothesis.py — _update_phase_a_shadow 
def _update_phase_a_shadow(self, snapshot, debug):
    s = self.state.shadow_thesis.phase_a
    _, a_facts = _ec_candidate(snapshot, "htf_pd_objective")

    # Exact HTF objective touch (sticky once set)
    if a_facts.get("phase_a_finale_touched") and not s.phase_a_objective_touched:
        s.phase_a_objective_touched = True
        s.phase_a_objective_touched_at = debug.get("cursor_time")

03 A.watch State Machine

A.watch has two internal states: A.watch (active pro attempt) and A.watch_weaken (counter structural move fired within A zone, conviction reduced). The 2-state design mirrors C's C.pullback / C.pullback_weaken exactly, but at the pro-attempt level.

State: A.watch

The active state. Each bar checks exit conditions in priority order.

# Priority 1 — E.seeking (new HTF extreme)
phase_e_context_new_htf_extreme == True
  OR e_ctx.get("new_htf_extreme") == True
→ E.seeking   # HTF continuation resumes; A.watch invalidated

# Priority 2 — A.watch_weaken (counter-HTF MSS — pro orderflow broke)
ltf_counter_orderflow_mss_watch == True   # MSS, NOT a structural SC
AND ltf_counter_orderflow_started_at > a_shadow.entered_at
→ A.watch_weaken
  a_shadow.pro_attempt_weaken    = True    # sticky
  a_shadow.pro_attempt_weaken_at = cursor_time

# Hold: neither exit fires → carry forward, update shadow
The A.watch_weaken trigger is an orderflow MSS, not a structural SC/ChoCh. An MSS means the pro orderflow itself broke — a more significant signal than a structural correction. A structural SC inside the B commitment range is not sufficient to weaken A.

State: A.watch_weaken

Entered when a counter-HTF MSS fires within the A zone. The pro attempt lost orderflow conviction but has not yet breached the B commitment extreme. Three exits in priority order.

# Priority 1 — E.seeking (new HTF extreme, highest priority)
phase_e_context_new_htf_extreme == True
→ E.seeking   # unexpected HTF reversal; entire thesis invalidated

# Priority 2 — C.pullback (B commitment extreme breached — B+A thesis invalidated)
current_price < b_shadow.commitment_extreme_level   # long setup
  OR current_price > b_shadow.commitment_extreme_level   # short setup
→ C.pullback
  c_shadow.origin_node = "A.watch_weaken_invalidated"   # enables 90% ceiling at next C→B

# Priority 3 — A.watch (recovery: pro-HTF SC BoS + pro extreme advance gate)
last_sc.breakDirection == pro-HTF   # pro structural SC (same gate as B→A entry)
AND last_sc.eventTimestamp > a_shadow.pro_attempt_weaken_at   # freshness floor
AND pro-side range extreme advanced past a_shadow.pro_extreme_at_weaken
→ A.watch
  a_shadow.recover_at = cursor_time
  # pro_attempt_weaken stays True (sticky — Layer 5 reads conviction history)
  # pro_extreme_at_weaken not updated — recovery reference remains from first weaken

# Hold
! B commitment extreme — b_shadow.commitment_extreme_level is ltf_struct.range_low (long) / range_high (short) captured at the C→B entry bar (not B→A). It cannot be queried live: ltf_struct.range_low ratchets on every new structural SC and will not retain B's original commitment extreme.

Recovery uses a pro-HTF SC + pro extreme advance gate. ltf_counter_orderflow_mss_watch is a counter-HTF signal — using it for recovery would fire on more counter pressure, not pro recovery. A pro-HTF last_sc BoS (same gate as B→A entry) is the correct signal. Additionally the pro extreme advance gate must pass — this prevents recovery when pro SCs fire within a sideways range with no genuine structural advance. Freshness floor is pro_attempt_weaken_at.
pro_attempt_weaken is sticky — it stays True even after recovery. Layer 5 reads this flag and raises the entry bar: requires phase_a_shadow_objective_touched == True or b_shadow.liquidity_pool_run == True before permitting entry on a recovered A.watch.

04 DAG Block — B → A

The B block checks for A entry after the HTF extreme check (highest priority) and before the counter MSS exit. If a fresh pro-HTF BoS is detected, shadow is initialized and the A block takes over from the next bar. The A block mirrors the B block structure exactly.

# (checked before counter MSS — see 402-hypothesis-phB.html §03)

# Exit 4 — C.pullback (counter MSS, freshness-guarded)
_mss_b       = e_ctx_b.get("ltf_counter_orderflow_mss_watch", False)
_mss_started = e_ctx_b.get("ltf_counter_orderflow_started_at")
if (_mss_b and _mss_started
        and str(_mss_started) > str(b_shadow.entered_at or "")):
    c_shadow = self.state.shadow_thesis.phase_c
    c_shadow.origin_node = "B.watch_mss_failed"
    c_shadow.entered_at  = _mss_started
    return self._commit(self._phase_c(direction, None, "watching", cursor_time,

# A block — called each bar when phase == "A"
a_shadow = self.state.shadow_thesis.phase_a
b_shadow = self.state.shadow_thesis.phase_b   # carries commitment_extreme_level (locked at C→B entry)
self._update_phase_a_shadow(snapshot, debug)
debug.update(self._phase_a_watch_shadow_debug())

_, e_ctx_a = _ec_candidate(snapshot, "phase_e_context")
# current_price from location_context.current_price (inside candidate, not debug_facts)
_cand_a, _ = _ec_candidate(snapshot, "phase_e_context")
_loc_a = (_cand_a.get("location_context") or {})

# Exit 1 — E.seeking (new HTF extreme, highest priority)
if bool(debug.get("phase_e_context_new_htf_extreme") or e_ctx_a.get("new_htf_extreme")):
    return self._commit(self._phase_e(direction, cursor_time, debug, ltf))

_mss_a       = e_ctx_a.get("ltf_counter_orderflow_mss_watch", False)
_mss_started_a = e_ctx_a.get("ltf_counter_orderflow_started_at")
_a_floor     = a_shadow.entered_at or ""
_b_level     = b_shadow.commitment_extreme_level   # B's commitment extreme (locked at C→B entry)
current_c_sub_a = debug.get("phase_sub_status", "watch")

if current_c_sub_a == "watch":
    # Weaken: counter-HTF MSS fires (pro orderflow broke)
    # Lock pro-side extreme reference for recovery gate at same time
    if (_mss_a and _mss_started_a and str(_mss_started_a) > str(_a_floor)):
        a_shadow.pro_attempt_weaken    = True
        a_shadow.pro_attempt_weaken_at = cursor_time
        _pro_extreme_raw = (ltf.get("range_high") if direction == "long" else ltf.get("range_low")) if ltf else None
        a_shadow.pro_extreme_at_weaken = float(_pro_extreme_raw) if _pro_extreme_raw is not None else None
        return self._commit(self._phase_a_watch(direction, cursor_time, debug,
            phase_sub_status="watch_weaken"))

elif current_c_sub_a == "watch_weaken":
    # Exit 2 — C.pullback (B commitment extreme breached — B+A invalidated)
    _cur_price_raw = _loc_a.get("current_price")
    if _b_level is not None and _cur_price_raw is not None:
        _cur_price = float(_cur_price_raw)
        commitment_extreme_breached = (
            _cur_price < float(_b_level) if direction == "long"
            else _cur_price > float(_b_level)
        )
        if commitment_extreme_breached:
            c_shadow = self.state.shadow_thesis.phase_c
            c_shadow.origin_node = "A.watch_weaken_invalidated"
            c_shadow.entered_at  = cursor_time
            return self._commit(self._phase_c(direction, None, "watching", cursor_time,
                {**debug, "phase_a_invalidated": True}, phase_sub_status="pullback"))

    # Recovery: pro-HTF SC BoS + pro extreme advance gate (no recovery if SC fires within same range)
    pro_break_a = "up" if direction == "long" else "down"
    _last_sc_a = (ltf.get("last_sc") or {}) if ltf else {}
    _last_sc_a_at = _last_sc_a.get("eventTimestamp")
    _weaken_floor = a_shadow.pro_attempt_weaken_at or _a_floor
    _pro_extreme_floor = a_shadow.pro_extreme_at_weaken
    _pro_extreme_advance_gate = True
    if _pro_extreme_floor is not None and ltf:
        _cur_pro_extreme_raw = ltf.get("range_high") if direction == "long" else ltf.get("range_low")
        if _cur_pro_extreme_raw is not None:
            _pro_extreme_advance_gate = (
                float(_cur_pro_extreme_raw) > _pro_extreme_floor if direction == "long"
                else float(_cur_pro_extreme_raw) < _pro_extreme_floor
            )
    if (_last_sc_a_at and str(_last_sc_a_at) > str(_weaken_floor)
            and _last_sc_a.get("breakDirection") == pro_break_a
            and _pro_extreme_advance_gate):
        a_shadow.recover_at = cursor_time
        return self._commit(self._phase_a_watch(direction, cursor_time, debug,
            phase_sub_status="watch"))

return self._commit(self._carry_current_hypothesis(cursor_time, debug))

05 Algorithm

Live implementation. Mirrors the B block pattern. See hypothesis.py:460–548 (A block) and hypothesis.py:2500–2545 (emit + shadow methods).

# hypothesis.py — _phase_a_watch emit method

def _phase_a_watch(
    self,
    direction: Direction,
    ts: str,
    debug: dict,
    phase_sub_status: str = "watch",
) -> Hypothesis:
    return Hypothesis(
        phase="A",
        status="watching",
        phase_sub_status=phase_sub_status,   # "watch" | "watch_weaken"
        poi_id=None,
        entry_policy="wait",
        target_policy="htf_pd_objective",
        direction=direction,
        debug_facts=debug,
    )
# hypothesis.py — _update_phase_a_shadow
# EC candidate "htf_pd_objective" emits phase_a_finale_touched / phase_a_finale_objective.

def _update_phase_a_shadow(
    self,
    snapshot: dict[str, Any],
    debug: dict,
) -> None:
    s = self.state.shadow_thesis.phase_a
    _, a_facts = _ec_candidate(snapshot, "htf_pd_objective")

    if (a_facts.get("phase_a_finale_touched")
            and not s.phase_a_objective_touched):
        s.phase_a_objective_touched = True
        s.phase_a_objective_touched_at = debug.get("cursor_time")
# hypothesis.py — _phase_a_watch_shadow_debug

def _phase_a_watch_shadow_debug(self) -> dict:
    s = self.state.shadow_thesis.phase_a
    return {
        "phase_a_shadow_entered_at":                s.entered_at,
        "phase_a_shadow_pro_attempt_weaken":        s.pro_attempt_weaken,
        "phase_a_shadow_pro_attempt_weaken_at":     s.pro_attempt_weaken_at,
        "phase_a_shadow_pro_extreme_at_weaken":          s.pro_extreme_at_weaken,   # pro extreme advance reference
        "phase_a_shadow_recover_at":                s.recover_at,
        "phase_a_shadow_objective_touched":         s.phase_a_objective_touched,
        "phase_a_shadow_objective_touched_at":      s.phase_a_objective_touched_at,
    }

06 Old Model — Archived

The pre-rework Phase A code was disabled before completion. It read from the htf_pd_objective EC candidate (internal pattern name "n") and had no sub-state machine. The B → A code path was commented out. All are superseded by the A.watch model.

_phase_a_setup (archived) hypothesis.py — dead, never completed

Entry: _phase_a_setup() read from the htf_pd_objective EC candidate (pattern name "n" in evidence_compiler.py). The EC emitted directional event labels:

bullish_phase_a_close_above_n
bullish_phase_a_touch_without_close_above_n
bearish_phase_a_close_below_n
bearish_phase_a_touch_without_close_below_n

These labels distinguished a confirmed close through the objective zone from a touch without close. The distinction was never consumed by the DAG — _phase_a_setup() was stubbed out and never called.

B → A path: commented out in hypothesis.py. No gate conditions were defined. The entry trigger, freshness floor, and sub-state machine were all unspecified.

Sub-statuses: none — Phase A was a single-state stub with no internal transitions documented.

Why archived: The A.watch model supersedes this with a complete 2-state machine (watch / watch_weaken), explicit B → A gate conditions, D/B-symmetric exits, and a pointer-only shadow (PhaseAShadow). The old EC event labels (close_above_n / touch_without_close) are Layer 5 territory — they inform entry quality, not DAG transitions.

Edge Case

A.watch re-entry after A → B failure. If A.watch → B.watch (failed) and B.watch then fires another pro-HTF BoS, A.watch opens again within the same epoch. Shadow is reset only on epoch boundary — pro_attempt_weaken from the prior A episode will carry over, keeping Layer 5 on elevated-bar mode for the re-entry.
